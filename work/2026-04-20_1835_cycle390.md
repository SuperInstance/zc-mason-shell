## Proposed Edge-Case Tests

### Test 1: Protocol Priority Violation (Skip P1)
**Scenario:** Agent attempts to execute P2 optimization without first establishing a P1 safe channel.
**Rationale:** The Deadband Protocol requires strict P0→P1→P2 progression. Skipping steps should be caught.
**Expected:** Room rejects the action with appropriate error, maintains P0 state.

### Test 2: Empty Action Space in P1
**Scenario:** After P0 mapping, the computed safe channel (P1) has zero valid actions (empty set).
**Rationale:** This is a degenerate case where negative space eliminates all options. The protocol must handle this without crashing.
**Expected:** Room transitions to a "deadlock" state or provides a specific error tile indicating no safe channels exist.

### Test 3: Maximum Boundary at P2 Optimization
**Scenario:** During P2 optimization, the computed "best path" hits an upper bound (e.g., maximum iterations, resource limit).
**Rationale:** Optimization algorithms can run indefinitely; the room should enforce limits.
**Expected:** Optimization terminates gracefully, returns best-found path up to limit, and produces a tile indicating boundary reached.

### Test 4: Malformed Tile Injection
**Scenario:** An agent submits a tile that does not conform to the expected schema (missing required fields, wrong data types).
**Rationale:** The room's tile validation logic must reject invalid tiles without corrupting internal state.
**Expected:** Tile is rejected, agent receives validation error, room state remains unchanged.

### Test 5: Concurrent Session Interference
**Scenario:** Two agents interact with the same DeadbandRoom instance simultaneously (simulated via rapid sequential calls).
**Rationale:** The room may not be designed for concurrency; we need to verify state isolation or identify race conditions.
**Expected:** Either each agent's session is isolated (thread-safe), or the room serializes requests. No cross-agent state contamination.
