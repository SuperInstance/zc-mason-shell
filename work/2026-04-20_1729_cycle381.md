## New Edge-Case Tests

### Test 1: Concurrent Tile Submission
**Scenario:** Simulate 10 agents submitting P0 tiles simultaneously.
**Edge:** Race conditions in tile queue, state corruption.
**Assertion:** All tiles are processed, final state is consistent, no data loss.

### Test 2: Malformed JSON Payload
**Scenario:** Agent sends a tile with invalid JSON (e.g., truncated, extra commas).
**Edge:** Parser robustness, error handling without crash.
**Assertion:** Room rejects tile with clear error, maintains operational state.

### Test 3: P0→P1→P0 Loop Detection
**Scenario:** Agent repeatedly submits P0 tiles after P1, violating strict priority.
**Edge:** Protocol enforcement, infinite loop prevention.
**Assertion:** Room logs warning, may throttle agent, but does not break.

### Test 4: Tile Payload Size Limit
**Scenario:** Agent submits a tile with 10MB base64-encoded image data.
**Edge:** Memory allocation, processing latency, storage limits.
**Assertion:** Room accepts but may truncate or compress; performance metrics recorded.

### Test 5: Simulated Network Partition During Write
**Scenario:** Tile write succeeds locally but fails to persist to backing store.
**Edge:** Data consistency, atomicity.
**Assertion:** Room enters safe mode, retries, or provides idempotent retry token.
