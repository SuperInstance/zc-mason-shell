## Edge-Case Tests Written

### Test 1: Multi-Agent Protocol Contention
**Scenario:** Two agents attempt P0 mapping simultaneously on overlapping negative space.  
**Check:** Protocol ensures no contradictory P0 tiles are accepted; later agent receives conflict signal.  
**Purpose:** Validate deadband room handles concurrent P0 submissions without corruption.

### Test 2: Invalid Priority Skip Detection
**Scenario:** Agent submits P2 optimization tile without completing P1 safe channel mapping.  
**Check:** Room rejects tile with "PriorityViolation" error, logs violation, agent state reset to P1.  
**Purpose:** Enforce strict P0→P1→P2 sequence; prevent protocol bypass.

### Test 3: Ghost Tile Integration
**Scenario:** Agent accesses P0 ghost tiles from previous failed agents.  
**Check:** Ghost tiles are read-only, cannot be modified, and are included in negative space map.  
**Purpose:** Ensure persistent P0 knowledge from dead agents is integrated but protected.

### Test 4: Maximum Channel Saturation
**Scenario:** Agent defines 255 safe channels (P1), then attempts to add one more.  
**Check:** Room returns "ChannelLimitExceeded"; agent must optimize within existing channels (P2).  
**Purpose:** Test system limits and graceful handling of boundary conditions.

### Test 5: Malformed Tile Payload
**Scenario:** Agent submits tile with invalid JSON structure and mismatched priority field.  
**Check:** Room rejects tile with "InvalidPayload" error, does not affect agent's protocol state.  
**Purpose:** Validate input sanitization and error resilience.

## Implementation Notes
- Tests follow existing pattern: `#[tokio::test]` async, use `DeadbandRoom::new()` and `room.process_tile()`.
- Each test isolates one edge condition.
- Assertions check both outcome (accept/reject) and side effects (state, logs).
- Mock agent IDs used: `agent_a`, `agent_b`.
