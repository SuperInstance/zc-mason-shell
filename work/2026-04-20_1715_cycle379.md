## New Edge-Case Tests

### Test 1: Protocol with Mixed Data Types (P0 Violation Handling)
**Scenario:** The `apply_deadband_protocol` receives a list containing non-numeric elements (strings, None). According to the protocol, P0 (negative space) should map invalid inputs.
**Test Focus:** Ensure the function either gracefully filters, converts, or raises a clear error—documenting the P0 constraint.
**Rationale:** Real-world data from fleet sensors or logs may be noisy.

### Test 2: Overlapping Safe Channels (P1 Ambiguity)
**Scenario:** `find_safe_channels` is given a sequence where candidate safe intervals overlap (e.g., thresholds produce channels [1,5] and [3,8]).
**Test Focus:** Verify the channel merging logic or priority resolution. The protocol states P1 is "find safe channels"—overlaps must be resolved deterministically.
**Rationale:** Ensures channel detection is robust for continuous or dense data.

### Test 3: Single-Element Channel Optimization (P2 Edge)
**Scenario:** `optimize_within_channels` receives a safe channel containing only one data point.
**Test Focus:** Confirm the optimization step doesn't crash and returns the single point (or handles it as already optimal). P2 optimization within a one-element channel should be a no-op.
**Rationale:** Channels may be narrow; optimization must be stable.

### Test 4: Extreme Threshold Values in Channel Detection
**Scenario:** `find_safe_channels` is called with a `threshold` parameter that is zero, negative, or extremely large (greater than max data range).
**Test Focus:** Validate behavior: zero threshold might mean no safe channels (all points are hazards?), negative threshold should be invalid (P0), large threshold might make the entire range a single channel.
**Rationale:** Parameter validation is a common source of bugs; aligns with P0 mapping invalid inputs.

### Test 5: Protocol with All Data in Negative Space (No Safe Channels)
**Scenario:** `apply_deadband_protocol` receives data where every point is a "rock" (e.g., all values are NaN, infinite, or outside any viable threshold).
**Test Focus:** The protocol should return an empty list or a sentinel indicating no safe path. P1 fails, so P2 cannot be reached.
**Rationale:** Tests the protocol's completeness—handling the case where no safe channel exists.
