## 5 Edge-Case Tests for DeadbandRoom

### Test 1: P0 Violation — Submitting Tile with P2 Content Before P1 Mapped
**Scenario:** Agent attempts to submit a tile containing P2 optimization logic without first establishing P1 safe channels.
**Expected Behavior:** Room rejects tile, logs P0 violation, increments `violation_count`. Tile not added to training set.
**Rationale:** Ensures strict adherence to Deadband Protocol priority.

### Test 2: Ghost Tile Injection with Malformed Metadata
**Scenario:** System injects a ghost tile (from dead agent) with missing or corrupted `lesson_type` field.
**Expected Behavior:** Room sanitizes tile, defaults `lesson_type` to "P0" if unreadable, logs warning. Tile added as P0 negative-space knowledge.
**Rationale:** Validates robustness against corrupted legacy data.

### Test 3: Concurrent Tile Submission from Multiple Agents
**Scenario:** Two agents simultaneously submit tiles for same room cycle.
**Expected Behavior:** Room uses locking mechanism (e.g., file lock, DB transaction) to serialize submissions. Both tiles processed without state corruption.
**Rationale:** Tests thread safety in multi-agent training environments.

### Test 4: Room Reset During Active Training
**Scenario:** Admin triggers `reset_room()` while agents are actively submitting tiles.
**Expected Behavior:** Room gracefully pauses submissions, archives current tiles to ghost storage, resets state, resumes. No tile loss.
**Rationale:** Ensures data integrity during maintenance operations.

### Test 5: Invalid Priority Transition (Skipping P1)
**Scenario:** Agent attempts to move from P0 directly to P2 in a single tile.
**Expected Behavior:** Room rejects transition, forces agent back to P1 mapping. Logs "Priority skip attempted".
**Rationale:** Enforces protocol's "Never skip to P2" rule.
