## 5 Additional Edge-Case Tests

### 1. Test: `test_submit_tile_with_ghost_flag`
**Scenario:** Submit a tile with `is_ghost=True` (simulating a dead agent's lesson).  
**Edge:** Ghost tiles should bypass normal agent validation but still require P0/P1/P2 structure.  
**Assert:** Tile is accepted, marked as ghost, and stored with `contributor="ghost"`.

### 2. Test: `test_concurrent_tile_submissions`
**Scenario:** Simulate two agents submitting tiles simultaneously.  
**Edge:** Room's `submit_tile` method should handle concurrent writes without data corruption.  
**Method:** Use `threading` to submit from two mock agents.  
**Assert:** Both tiles are stored, order is preserved (or handled via timestamp), no exceptions.

### 3. Test: `test_malicious_p2_skip_attempt`
**Scenario:** Agent submits a tile that claims P2 optimization but lacks prior P0/P1 steps.  
**Edge:** Room should detect protocol violation even if tile structure is valid.  
**Assert:** Tile is rejected with error "P0/P1 not satisfied before P2".

### 4. Test: `test_room_reset_after_max_capacity`
**Scenario:** Room reaches configured max tile capacity (e.g., 1000 tiles).  
**Edge:** Next submission should trigger archival and reset, not crash.  
**Assert:** Old tiles are archived (mocked), room state resets, new tile is stored.

### 5. Test: `test_invalid_sentiment_field`
**Scenario:** Tile includes `sentiment` field with out-of-bounds value (e.g., `-5` or `+10`).  
**Edge:** Sentiment should be clamped to [-1, +1] or rejected.  
**Assert:** Tile is either accepted with clamped sentiment or rejected with validation error.
