## 1. Reading the DeadbandRoom Preset

I examined the actual file `plato-torch/src/presets/deadband_room.py` to understand its structure and behavior. Key findings:

**Architecture:**
- DeadbandRoom inherits from `BaseTrainingRoom`
- Implements the Deadband Protocol (P0, P1, P2 priority system)
- Maintains state: `negative_space` (P0), `safe_channels` (P1), `optimized_paths` (P2)
- Core method: `process_observation(observation)` returns action

**Key Methods:**
- `_map_negative_space()`: Identifies prohibited actions
- `_find_safe_channels()`: Finds permissible action spaces  
- `_optimize_within_channels()`: Selects optimal action from safe set
- `_validate_protocol()`: Ensures P0→P1→P2 priority order

**Edge Cases Observed:**
- Empty observation handling
- Conflicting constraints in negative space
- Overlapping safe channels
- No safe channels found (deadlock)
- Protocol validation failures

"""
Edge-case tests for DeadbandRoom preset.
Tests pathological conditions and protocol violations.
"""

import pytest
import numpy as np
from plato_torch.src.presets.deadband_room import DeadbandRoom


class TestDeadbandRoomEdgeCases:
    
    def test_empty_observation_handling(self):
        """P0: Empty observation should not crash, should return safe default."""
        room = DeadbandRoom()
        observation = {}
        
        # Should handle gracefully
        action = room.process_observation(observation)
        
        # Should have mapped some negative space (even if empty)
        assert hasattr(room, 'negative_space')
        # Should return a valid action dict
        assert isinstance(action, dict)
    
    def test_conflicting_negative_space(self):
        """P0: Conflicting constraints in negative_space should resolve deterministically."""
        room = DeadbandRoom()
        
        # Simulate conflicting P0 constraints
        room.negative_space = {
            'action_type': ['move_left', 'move_right'],  # Can't move either way?
            'target': ['enemy', 'ally']  # Can't target anyone?
        }
        
        # Force P1 to find channels despite contradictions
        room.safe_channels = room._find_safe_channels()
        
        # Protocol should handle this, not crash
        assert room.safe_channels is not None
        # Should log conflict for review
    
    def test_no_safe_channels_deadlock(self):
        """P1: When no safe channels exist, protocol should handle deadlock."""
        room = DeadbandRoom()
        
        # Create impossible negative space that blocks all actions
        room.negative_space = {'all_actions': True}
        room.safe_channels = room._find_safe_channels()
        
        # With no safe channels, what happens?
        if len(room.safe_channels) == 0:
            # Should have deadlock handling
            assert hasattr(room, '_handle_deadlock')
            result = room._handle_deadlock()
            assert result is not None  # Some recovery action
        else:
            # If channels exist, ensure they're truly safe
            for channel in room.safe_channels:
                assert not self._violates_negative_space(channel, room.negative_space)
    
    def test_overlapping_safe_channels(self):
        """P1: Overlapping safe channels should not cause duplicate optimization."""
        room = DeadbandRoom()
        
        # Create overlapping channels
        room.safe_channels = [
            {'action': 'A', 'param': 1},
            {'action': 'A', 'param': 2},  # Same action, different param
            {'action': 'B', 'param': 1}
        ]
        
        # Optimization should handle duplicates
        optimized = room._optimize_within_channels()
        
        # Should select one action, not crash on duplicates
        assert optimized is not None
        # Should be from safe_channels
        assert any(self._actions_equal(optimized, channel) for channel in room.safe_channels)
    
    def test_protocol_validation_failure(self):
        """P0→P1→P2: Violating priority order should be detected."""
        room = DeadbandRoom()
        
        # Simulate protocol violation (skipping to P2)
        room.optimized_paths = [{'action': 'skip_ahead'}]  # P2 without P0/P1
        
        # Validation should catch this
        is_valid, error = room._validate_protocol()
        
        assert not is_valid
        assert 'priority' in error.lower() or 'protocol' in error.lower()
        # Should provide actionable error message
    
    # Helper methods
    def _violates_negative_space(self, action, negative_space):
        """Check if action violates any negative space constraint."""
        for key, forbidden in negative_space.items():
            if key in action and action[key] in forbidden:
                return True
        return False
    
    def _actions_equal(self, a, b):
        """Compare two action dictionaries."""
        return all(a.get(k) == b.get(k) for k in set(a.keys()) | set(b.keys()))

## 3. Test Rationale (P0→P1→P2 Analysis)

**P0 (Negative Space - What NOT to test):**
- Don't test implementation details that may change
- Don't assume specific error messages
- Don't test random or non-deterministic behavior

**P1 (Safe Channels - What CAN be tested):**
- Empty/edge inputs should not crash
- Conflicting constraints should resolve deterministically  
- Protocol violations should be detectable
- Deadlock conditions should have recovery paths
- Overlapping options should not break optimization

**P2 (Optimization - Best test approach):**
- Tests should be isolated and independent
- Each test validates one specific edge case
- Use helper methods to reduce duplication
- Focus on behavior, not implementation
