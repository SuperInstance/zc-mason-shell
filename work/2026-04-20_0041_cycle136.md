## Edge-Case Test Proposals (5 New Tests)

### 1. **Test: `test_p0_violation_with_empty_negative_space`**
- **Scenario:** Negative space (P0 map) is provided as an empty list.
- **Expected Behavior:** Protocol should treat empty negative space as "no restrictions" and proceed to P1.
- **Risk:** Code might assume non-empty list and crash.
- **P0 Relevance:** Ensures agent doesn't crash when there are no predefined "rocks."

### 2. **Test: `test_p1_no_safe_channels`**
- **Scenario:** After P0, all possible actions are blocked (no safe channels).
- **Expected Behavior:** Protocol should raise a `NoSafeChannelError` (or equivalent) and not proceed to P2.
- **Risk:** Infinite loop or silent failure if code assumes at least one safe channel exists.
- **P1 Relevance:** Maps the negative space of having zero safe paths.

### 3. **Test: `test_p2_optimize_with_single_channel`**
- **Scenario:** P1 yields exactly one safe channel (list of length 1).
- **Expected Behavior:** P2 should still run optimization on that single channel, returning it (or a refined version).
- **Risk:** Optimization logic might skip if `len(channels) == 1`, missing potential refinement.
- **P2 Relevance:** Tests optimization boundary condition.

### 4. **Test: `test_state_persistence_across_cycles`**
- **Scenario:** Simulate multiple consecutive calls to `run_protocol()` with incremental state.
- **Expected Behavior:** P0 map should accumulate across cycles (negative space grows), affecting later P1/P2.
- **Risk:** State might reset incorrectly between cycles, violating the "accumulate lessons" principle.
- **Architecture Relevance:** Aligns with ghost tiles and persistent P0 knowledge.

### 5. **Test: `test_malformed_action_object`**
- **Scenario:** An action in the candidate set is `None` or has missing required fields.
- **Expected Behavior:** Protocol should filter out invalid actions during P0/P1, not crash.
- **Risk:** Type errors or attribute errors could propagate.
- **Robustness:** Ensures the room handles malformed input from other fleet components.
