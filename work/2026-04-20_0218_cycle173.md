## Analysis of Existing Tests
The DeadbandRoom implements the fleet's P0/P1/P2 protocol. Current tests in `test_deadband.py` cover:
- Basic protocol flow (P0 → P1 → P2)
- Valid state transitions
- Error handling for invalid transitions (e.g., skipping to P2)
- Tile submission validation

## Five Additional Edge-Case Tests

### 1. Test: Empty Negative Space (P0 with zero constraints)
**Scenario:** Agent submits P0 mapping with empty list (no constraints).
**Expected Behavior:** Room should accept empty P0 as valid (agent sees no rocks) and transition to P1.
**Rationale:** Tests protocol handling of "no negative space" edge case.

### 2. Test: P1 Channel Exhaustion (All channels blocked by P0)
**Scenario:** P0 defines constraints that block all possible P1 channels.
**Expected Behavior:** Room should reject P1 submission with "no safe channels" error, preventing transition to P2.
**Rationale:** Tests dead-end scenario where agent cannot proceed beyond P1.

### 3. Test: Concurrent Tile Submission Race Condition
**Scenario:** Simulate two agents submitting tiles to same room simultaneously.
**Expected Behavior:** Room should handle concurrent writes without data corruption (e.g., using locks or atomic operations).
**Rationale:** Tests thread safety in multi-agent environments.

### 4. Test: Malformed Tile JSON Structure
**Scenario:** Agent submits tile with missing required fields or invalid JSON.
**Expected Behavior:** Room should reject submission with descriptive error, not crash.
**Rationale:** Tests robustness against malformed input.

### 5. Test: Protocol Loop Detection (P2 → P0 regression)
**Scenario:** Agent attempts to return to P0 after reaching P2.
**Expected Behavior:** Room should reject regression with "protocol violation" error, maintaining P2 state.
**Rationale:** Tests strict forward-only protocol enforcement.

4. Clean up test artifacts
