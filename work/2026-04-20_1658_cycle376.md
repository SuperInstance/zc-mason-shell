## 5 Additional Edge-Case Tests
Following fleet doctrine (P0 first), I'll write tests for the highest-risk gaps. I'll append these to `plato-torch/tests/test_deadband_room.py`.

### Test 1: Tile with Malformed Content
**Scenario:** Tile `content` is `None` instead of a dict.
**Expected:** Room should handle gracefully—likely log error and skip P0/P1/P2 processing.
**Risk:** Could cause `AttributeError` or `TypeError` if code assumes `content` is dict.

### Test 2: Tile Missing Protocol Fields
**Scenario:** Tile `content` is a dict but missing `p0`, `p1`, or `p2` keys.
**Expected:** Room should treat missing fields as empty lists (or default). Should not crash.
**Risk:** Could cause `KeyError` if code accesses keys directly without `.get()`.

### Test 3: Extreme Numeric Values in Tile
**Scenario:** Tile has extremely large positive/negative scores, or `NaN`/`inf` values.
**Expected:** Room's threshold comparisons should handle gracefully (not crash or produce invalid comparisons).
**Risk:** Floating-point errors, infinite loops, or comparison failures.

### Test 4: `export_ensign` with Disk Full
**Scenario:** Simulate `OSError` (e.g., disk full) when saving adapter.
**Expected:** Room should catch `OSError` and raise a descriptive `RoomExportError`.
**Risk:** Unhandled exception could crash the training pipeline.

### Test 5: Invalid Room Initialization
**Scenario:** Initialize `DeadbandRoom` with invalid parameters (e.g., `room_id=-1`, `name=None`).
**Expected:** Should raise `ValueError` or `TypeError` at initialization.
**Risk:** Invalid state could cause subtle bugs later.

def test_process_tile_malformed_content(self):
    """Test DeadbandRoom.process_tile with content=None."""
    room = DeadbandRoom(room_id=1, name="test_room")
    tile = {"tile_id": "bad_tile", "content": None}
    # Should not raise exception
    result = room.process_tile(tile)
    # Expect the tile to be marked invalid or skipped
    assert "error" in result or result.get("processed") is False

def test_process_tile_missing_protocol_fields(self):
    """Test tile missing p0/p1/p2 keys."""
    room = DeadbandRoom(room_id=1, name="test_room")
    tile = {"tile_id": "incomplete_tile", "content": {"p0": [], "p1": []}}  # missing p2
    result = room.process_tile(tile)
    # Should still process with default empty p2
    assert "p2" in result.get("content", {})
    assert result["content"]["p2"] == []

def test_process_tile_extreme_scores(self):
    """Test tile with extreme numeric values."""
    room = DeadbandRoom(room_id=1, name="test_room")
    tile = {
        "tile_id": "extreme_tile",
        "content": {
            "p0": [{"score": float("inf")}],
            "p1": [{"score": -float("inf")}],
            "p2": [{"score": float("nan")}]
        }
    }
    result = room.process_tile(tile)
    # Should not crash; handling of inf/nan is implementation-specific
    assert "content" in result
    # Ensure no exception was raised

def test_export_ensign_disk_full(self, mocker):
    """Test export_ensign when disk is full."""
    room = DeadbandRoom(room_id=1, name="test_room")
    mock_save = mocker.patch("peft.LoraConfig.save_pretrained")
    mock_save.side_effect = OSError("No space left on device")
    with pytest.raises(RoomExportError, match="Failed to save adapter"):
        room.export_ensign("/fake/path")

def test_invalid_initialization(self):
    """Test DeadbandRoom initialization with invalid parameters."""
    with pytest.raises(ValueError):
        DeadbandRoom(room_id=-1, name="test_room")
    with pytest.raises(TypeError):
        DeadbandRoom(room_id=1, name=None)
