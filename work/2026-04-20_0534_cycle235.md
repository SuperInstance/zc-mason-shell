## P2 Optimization (Best Path — 5 Edge-Case Tests)
Based on the existing test suite, here are 5 additional edge-case tests:

### Test 1: Tile with Malformed Priority Field
**Scenario:** Agent submits a tile where the priority field is not a valid P0/P1/P2 string.
**Expected:** Room rejects tile, increments `invalid_tiles` counter, returns appropriate error.
**Rationale:** Ensures robust validation against protocol violations.

### Test 2: Deadband Threshold at Exact Boundary
**Scenario:** Configure deadband threshold to 0.5. Submit a tile with sentiment score of exactly 0.5.
**Expected:** Tile is rejected (deadband is inclusive? exclusive?). Need to check implementation.
**Rationale:** Tests boundary condition handling in deadband logic.

### Test 3: Rapid-Fire Tile Submission from Same Agent
**Scenario:** Simulate 1000 tiles submitted in quick succession from a single agent ID.
**Expected:** Room processes all tiles, maintains correct tile counts, does not crash or leak memory.
**Rationale:** Stress tests tile accumulation and state management.

### Test 4: Protocol Regression After P2 Tile
**Scenario:** Agent submits valid P0, then P1, then P2 tiles, then attempts to submit another P0 tile.
**Expected:** Room accepts the P0 tile? Or rejects it as protocol regression? Depends on spec.
**Rationale:** Tests whether protocol enforces monotonic progression or allows revisiting earlier phases.

### Test 5: Tile with Missing Required Fields
**Scenario:** Agent submits a tile missing the `agent_id` or `timestamp` fields.
**Expected:** Room rejects tile, does not increment `valid_tiles` counter.
**Rationale:** Ensures validation catches incomplete data before processing.
