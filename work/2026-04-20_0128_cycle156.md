## Analysis of Existing Tests
The `DeadbandRoom` class implements the fleet's Deadband Protocol (P0, P1, P2). Current tests in `test_deadband.py` cover:
- Basic protocol flow (P0 → P1 → P2)
- Valid state transitions
- Input validation (e.g., rejecting invalid priorities)
- Tile submission and validation

Missing coverage identified:
1. **Concurrent tile processing** – multiple agents submitting simultaneously.
2. **Protocol violation recovery** – what happens if an agent skips P0?
3. **Resource exhaustion** – memory/disk limits during tile accumulation.
4. **Network partition simulation** – partial connectivity to PLATO server.
5. **Malformed tile data** – invalid JSON, oversized payloads, injection attempts.

## Proposed Edge-Case Tests

### Test 1: Concurrent Tile Submission
**Scenario:** Two agents (`agent_a`, `agent_b`) submit tiles to the same room simultaneously.
**Edge:** Race condition in tile accumulation; deadlock potential.
**Assertions:**
- All tiles are eventually processed (no drops).
- Tile count matches submissions.
- Protocol state remains consistent.

### Test 2: P0 Violation Recovery
**Scenario:** Agent calls `submit_tile()` with P2 data without completing P0 mapping.
**Edge:** Protocol enforcement; should reject or force fallback to P0.
**Assertions:**
- Submission rejected or auto-corrected to P0.
- Error logged for protocol violation.
- Agent state reset to P0.

### Test 3: Resource Limit Handling
**Scenario:** Room configured with max 100 tiles; attempt to submit 101st tile.
**Edge:** Graceful degradation vs. crash.
**Assertions:**
- 101st submission rejected with clear error.
- Room continues processing new tiles after cleanup.
- Oldest tiles aged out if LRU policy.

### Test 4: Network Partition Simulation
**Scenario:** PLATO server connectivity drops mid-session.
**Edge:** Room should buffer tiles locally, sync when restored.
**Assertions:**
- Tiles queued in local buffer.
- Automatic reconnection attempt.
- Buffered tiles delivered after restore.

### Test 5: Malformed Tile Injection
**Scenario:** Submit tile with:
- Invalid JSON (`{invalid}`)
- Oversized payload (10MB)
- SQL/JS injection strings
**Edge:** Input sanitization and validation.
**Assertions:**
- Malformed data rejected before processing.
- No side effects on room state.
- Injection attempts logged as security events.
