## Analysis of Existing Test Suite
The existing `test_deadband_room.py` contains 12 tests covering:
- Basic room initialization and agent joining
- P0/P1/P2 protocol adherence
- Tile submission and validation
- Error handling for invalid tiles
- Room capacity limits

The tests use pytest fixtures for room setup and mock agents. Coverage appears strong for happy paths and common failure modes, but edge cases around concurrency, state persistence, and protocol boundary conditions are not fully explored.

## Proposed Edge-Case Tests
Based on the Deadband Protocol (P0→P1→P2 strict priority) and room behavior observed in the code, I've identified five edge cases not covered by existing tests:

### 1. **Test: Concurrent Tile Submission with Protocol Violation**
**Scenario:** Multiple agents submit tiles simultaneously where one violates P0 (negative space) while another follows P1 (safe channel).  
**Expected:** Room should reject violating tile, accept valid tile, maintain consistent state.  
**Rationale:** Tests thread safety and protocol enforcement under load.

### 2. **Test: Agent Rejoining After Protocol Ban**
**Scenario:** Agent gets banned for repeated P0 violations, leaves, attempts to rejoin with same ID.  
**Expected:** Room should reject rejoining or place agent in probationary state.  
**Rationale:** Tests ban persistence and rejoin logic.

### 3. **Test: P1→P2 Transition Without Complete P1 Coverage**
**Scenario:** Agent attempts P2 optimization while P1 safe channels are partially mapped (incomplete).  
**Expected:** Room should reject P2 tile, return to P1 mapping requirement.  
**Rationale:** Tests strict protocol priority enforcement.

### 4. **Test: Maximum Tile Size Boundary**
**Scenario:** Agent submits tile at exact size limit (config.MAX_TILE_SIZE).  
**Expected:** Tile should be accepted without truncation or error.  
**Rationale:** Tests boundary condition handling in validation logic.

### 5. **Test: Room State Persistence Across Serialization**
**Scenario:** Save room state, reset room, load state, verify protocol continuity.  
**Expected:** Agents' P0/P1/P2 progress should be restored accurately.  
**Rationale:** Tests state serialization for training checkpointing.
