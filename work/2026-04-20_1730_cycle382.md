## Identified Edge Cases (P0 Negative Space)
From reading the `deadband_room.py` implementation, I identified these untested scenarios:

1. **Concurrent tile submission** – Race condition when multiple agents submit simultaneously.
2. **Validator intermittent failure** – `TileValidator.validate()` raises an unexpected exception.
3. **Storage rollback on error** – `RoomStore.add_tile()` succeeds but subsequent step fails.
4. **Agent state persistence** – Agent marked `blocked` but room restarts (state reload).
5. **Protocol violation cascade** – Agent submits P2 tile without prior P0/P1, then corrects.

### Test 1: Concurrent Tile Submission
```python
import asyncio
from unittest.mock import AsyncMock
import pytest

@pytest.mark.asyncio
async def test_concurrent_tile_submission_does_not_exceed_capacity(deadband_room, sample_agent):
    """Multiple agents submitting tiles simultaneously should not exceed room capacity."""
    room, validator, store = deadband_room
    store.get_room_usage.return_value = {"capacity": 10, "current": 5}
    validator.validate.return_value = True
    
    # Create multiple tiles from different agents
    tiles = []
    for i in range(5):
        agent_id = f"agent_{i}"
        tile = {
            "agent_id": agent_id,
            "protocol_phase": "P0",
            "content": f"Test content {i}",
            "timestamp": "2026-04-20T17:30:00Z"
        }
        tiles.append(tile)
    
    # Submit all tiles concurrently
    tasks = [room.submit_tile(tile) for tile in tiles]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    
    # All should succeed without capacity violation
    assert all(r is True for r in results)
    assert store.add_tile.call_count == 5
```
