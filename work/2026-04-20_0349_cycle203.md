## 5 Edge-Case Tests for DeadbandRoom

### Test 1: `test_p0_violation_skips_to_p2`
**Scenario:** Agent submits a tile with P2 optimization data but no P0 negative space mapping.
**Expected:** Room rejects tile or marks as invalid. Protocol enforcement error.
**Rationale:** Strict priority — cannot skip to P2 without P0 and P1.

### Test 2: `test_empty_p1_channel_list`
**Scenario:** Agent correctly maps P0 negatives but provides an empty list for P1 safe channels.
**Expected:** Room should handle gracefully — either treat as dead-end (no safe path) or require P1 retry.
**Rationale:** P1 is "where you CAN be". Empty list means no viable channels — a valid but critical state.

### Test 3: `test_malformed_tile_json_structure`
**Scenario:** Tile JSON is missing required fields (`phase`, `priorities`, `content`) or has mismatched types.
**Expected:** Room raises validation error, logs malformed tile, does not crash.
**Rationale:** Robustness against malformed input from new/greenhorn agents.

### Test 4: `test_concurrent_tile_submission_race`
**Scenario:** Two agents submit tiles for the same room cycle simultaneously.
**Expected:** Room handles concurrency — either sequential processing or conflict detection.
**Rationale:** Fleet has multiple agents; room must manage concurrent submissions.

### Test 5: `test_ghost_tile_integration`
**Scenario:** Room receives a tile marked as `ghost: true` (from a dead agent).
**Expected:** Room accepts tile but treats it as P0-only knowledge (negative space lessons).
**Rationale:** Ghost tiles persist dead agents' lessons — room should integrate them without full protocol validation.
