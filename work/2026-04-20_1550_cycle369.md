## Edge-Case Test Proposals
Based on the Deadband Protocol's strict priority enforcement and the room's role in training, I propose these 5 edge-case tests:

### 1. **Test: P0-P1-P2 Priority Violation**
**Scenario:** Agent attempts P2 optimization before completing P0 and P1.
**Expected:** Room rejects the tile or logs a protocol violation. The Deadband Protocol states "Strict priority. Never skip to P2."
**Implementation:** Mock an agent submitting a P2 tile when P0 mapping is incomplete.

### 2. **Test: Channel Boundary Overlap**
**Scenario:** Safe channel (P1) boundaries intersect with negative space (P0).
**Expected:** Room should detect the conflict and either adjust channel boundaries or flag as invalid. A safe channel cannot include negative space.
**Implementation:** Create test data where P0 obstacles overlap with proposed P1 channel edges.

### 3. **Test: High-Velocity Tile Submission**
**Scenario:** Agent submits tiles faster than the room's processing rate.
**Expected:** Room implements queueing or rate limiting without dropping tiles. Training continuity should be maintained.
**Implementation:** Use threading to simulate rapid tile submissions and verify all are processed.

### 4. **Test: Partial Tile Data (Malformed JSON)**
**Scenario:** Agent submits a tile with missing required fields (e.g., phase, priority, content).
**Expected:** Room validates tile structure and rejects malformed submissions with descriptive error.
**Implementation:** Submit tiles missing 'phase' field, with invalid priority values, or with non-JSON content.

### 5. **Test: Concurrent Multi-Agent Channel Discovery**
**Scenario:** Multiple agents simultaneously discover different safe channels in the same space.
**Expected:** Room should handle concurrent writes, potentially merging channels or detecting conflicts. No data corruption.
**Implementation:** Use pytest's `threading` or `asyncio` to simulate simultaneous P1 submissions.

- Each test should be isolated and not depend on external services.
- Mock the tile storage backend to ensure test reliability.
