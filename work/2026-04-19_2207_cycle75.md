## Edge-Case Tests for DeadbandRoom

### Test 1: **P0 Violation – Attempting P2 Optimization Without Safe Channel**
**Scenario:** A training tile submits a P2 ("optimize") action before any P1 ("safe channel") has been established.
**Expected Behavior:** The room should reject the tile, log a P0 violation, and not advance the agent's phase.
**Test Logic:**
```python
def test_p2_before_p1_rejected():
    room = DeadbandRoom()
    # Simulate initial state with no P1 mapping
    room.current_phase = "P0"
    room.safe_channels = []
    
    tile = TrainingTile(action="optimize", phase="P2", content="try to optimize path")
    result = room.submit_tile(tile)
    
    assert result["accepted"] == False
    assert "P0 violation" in result["reason"]
    assert room.current_phase == "P0"  # Phase should not change
```

### Test 4: **Maximum Tile Capacity Boundary**

**Expected Behavior:** The room should accept tiles up to capacity, reject the excess, and possibly trigger a batch process or export.
**Test Logic:**
```python
def test_tile_capacity_enforced():
    room = DeadbandRoom(max_tiles=100)
    
    # Fill to capacity
    for i in range(100):
        tile = TrainingTile(action="map", phase="P0", content=f"tile_{i}")

    assert result["accepted"] == False
    assert "capacity" in result["reason"].lower() or "full" in result["reason"].lower()
```
