## Edge-Case Tests for DeadbandRoom

### Test 1: Empty Tile Submission (P0 Violation)
**Scenario:** An agent submits a tile with empty content.
**Expected:** Room should reject the tile, increment P0 violation counter, and not change state.
**Rationale:** Empty tiles provide no training value and waste processing cycles.

```python
def test_submit_empty_tile(self):
    """P0: Empty tile content should be rejected."""
    room = DeadbandRoom()
    initial_p0_count = room.p0_violations
    result = room.submit_tile(agent_id="test_agent", content="", priority=1)
    assert result["accepted"] is False
    assert "empty" in result["reason"].lower()
    assert room.p0_violations == initial_p0_count + 1
    assert len(room.tiles) == 0
```

### Test 2: Priority Skipping (P0 → P2 Direct)
**Scenario:** Agent attempts to submit P2 tile when P0 mapping is incomplete (<90% coverage).
**Expected:** Room should reject P2 tile, log P0 violation, maintain P0 requirement.
**Rationale:** Strict priority enforcement is core doctrine—never skip to P2.

```python
def test_priority_skip_p0_to_p2(self):
    """P0: Attempting P2 before P0 complete should fail."""
    room = DeadbandRoom()

    room.p0_coverage = 0.5
    room.current_phase = "P0"

```

### Test 3: Maximum Tile Size Boundary
**Scenario:** Agent submits tile at exact size limit (config.MAX_TILE_SIZE).
**Expected:** Tile should be accepted if valid, rejected if exceeds by even 1 byte.
**Rationale:** Boundary testing ensures robust size validation.

```python
def test_tile_size_boundary(self):
    """P1: Test exact maximum tile size and one byte over."""
    room = DeadbandRoom()
    max_size = room.config.MAX_TILE_SIZE
    
    # Exact size should work
    exact_content = "x" * max_size
    result1 = room.submit_tile(agent_id="test_agent", content=exact_content, priority=1)
    assert result1["accepted"] is True
    
    # One byte over should fail
    overflow_content = "x" * (max_size + 1)
    result2 = room.submit_tile(agent_id="test_agent", content=overflow_content, priority=1)
    assert result2["accepted"] is False
    assert "size" in result2["reason"].lower()
```

### Test 4: Concurrent Tile Submission Race

**Expected:** Room should handle concurrency, maintain consistent state, prevent double-counting.
**Rationale:** Real fleet usage involves concurrent agents; state must remain valid.

def test_concurrent_phase_transition(self):
    """P2: Concurrent submissions during P1→P2 transition."""
    room = DeadbandRoom()
    room.p1_coverage = 0.89  # Just below P1 threshold (90%)
    room.current_phase = "P1"
    
    # Simulate concurrent submissions
    def submit_p1_tile():

### Test 5: Malformed Priority Value
**Scenario:** Agent submits tile with priority=0 (invalid) and priority=4 (out of range).
**Expected:** Room should reject invalid priorities, log P0 violation for protocol breach.
**Rationale:** Input validation prevents corrupted state; only priorities 1-3 are valid.

```python
def test_invalid_priority_values(self):
    """P0: Priority values outside 1-3 should be rejected."""
    room = DeadbandRoom()
    
    # Priority 0 (invalid)
    result1 = room.submit_tile(agent_id="test_agent", content="test", priority=0)
    assert result1["accepted"] is False
    assert "priority" in result1["reason"].lower()
    
    # Priority 4 (out of range)
    result2 = room.submit_tile(agent_id="test_agent", content="test", priority=4)
    assert result2["accepted"] is False
    assert "priority" in result2["reason"].lower()
    
    # Valid priorities should work
    for p in [1, 2, 3]:
        result = room.submit_tile(agent_id="test_agent", content=f"P{p} test", priority=p)
        assert result["accepted"] is True
```
