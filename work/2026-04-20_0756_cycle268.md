## New Edge-Case Tests (5)

### 1. Test: `test_concurrent_agent_tile_submission`
**Scenario:** Simulate two agents submitting tiles simultaneously to the same DeadbandRoom instance.
**Edge Case:** Race condition on room's tile ledger or state corruption.
**Assertions:**
- All tiles are recorded (no drops).
- Tile IDs remain unique.
- Room state (phase, channel) remains consistent.

### 2. Test: `test_malformed_tile_rejection`
**Scenario:** Submit tiles with invalid structure: missing required fields, extreme numeric values, oversized payload, or embedded escape sequences.
**Edge Case:** Adversarial or buggy agent sending malformed data.
**Assertions:**
- Room rejects invalid tiles with appropriate error (not crash).
- Valid tiles submitted after rejection are still accepted.
- No side effects on room's internal state.

### 3. Test: `test_resource_exhaustion_handling`
**Scenario:** Simulate rapid submission of a large number of tiles (e.g., 10,000) or extremely large tile payloads.
**Edge Case:** File descriptor exhaustion, memory spike, or disk full.
**Assertions:**
- Room throttles or queues submissions gracefully.
- System resources (simulated) are monitored; room enters a safe state (e.g., pauses ingestion) rather than crashing.
- After cleanup, room can accept new tiles.

### 4. Test: `test_persistence_layer_failure`
**Scenario:** Mock the tile storage backend (e.g., database or filesystem) to fail on write.
**Edge Case:** Network partition, disk full, permission denied.
**Assertions:**
- Room detects persistence failure.
- Tile submission returns a clear error (not silent failure).
- Room remains operable for new attempts after failure is resolved (mock recovery).

### 5. Test: `test_partial_protocol_adherence_edge`
**Scenario:** Agent correctly maps P0 negative space but then misidentifies P1 safe channels (e.g., returns empty list). Or agent finds P1 channels but applies invalid optimization in P2.
**Edge Case:** Logical errors in agent implementation that still technically follow protocol steps.
**Assertions:**
- Room detects invalid P1 output (empty safe channels) and does not progress to P2.
- Room detects P2 optimization that violates P1 constraints and rejects the result.
- Appropriate warnings or error states are logged.
