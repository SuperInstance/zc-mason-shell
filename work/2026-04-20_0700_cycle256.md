## 5 Additional Edge-Case Tests

### Test 1: Empty Constraint Sets
**Scenario:** What happens when P0, P1, or P2 constraint sets are empty?
**Rationale:** Real-world initialization might have missing or incomplete constraint definitions.
**Test Focus:** Room should handle empty sets gracefully without crashing, possibly defaulting to permissive behavior for empty P0/P1.

### Test 2: Overlapping P0 and P1 Constraints
**Scenario:** A constraint appears in both P0 (negative space) and P1 (safe channels).
**Rationale:** Contradictory definitions could occur from conflicting rule sources.
**Test Focus:** Priority resolution—P0 should take precedence (deadband protocol), making the channel unsafe despite P1 listing.

### Test 3: Maximum Tile Size Boundary
**Scenario:** Submit a tile at the exact maximum allowed size (e.g., 1MB if limited).
**Rationale:** Boundary conditions often reveal off-by-one errors or buffer overflows.
**Test Focus:** Room acceptance/rejection logic at size limits, ensuring no truncation or corruption.

### Test 4: Rapid-Fire Tile Submissions
**Scenario:** Send multiple tiles in quick succession (burst of 10+ within a second).
**Rationale:** Tests concurrency handling, rate limiting, and state corruption under load.
**Test Focus:** Order preservation, rejection of excess tiles if throttled, and consistent room state after burst.

### Test 5: Malformed Tile Metadata
**Scenario:** Tile has missing required fields (e.g., no `agent_id`, invalid `timestamp`, malformed `content`).
**Rationale:** Robustness against malformed inputs from buggy or adversarial agents.
**Test Focus:** Proper validation and rejection with descriptive error, not silent failure or crash.

## Implementation Notes
- Each test should be isolated (fresh room instance).
- Use pytest.mark.parametrize for data-driven variants.
- Mock external dependencies (e.g., tile storage) to focus on room logic.
- Assert both behavior (accept/reject) and side effects (state changes, logs).

## P0 Consideration
These tests map negative space: what the room should NOT do (crash, corrupt state, accept invalid tiles). This aligns with Deadband Protocol P0.
