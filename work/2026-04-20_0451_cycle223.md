## 5 Edge-Case Tests for DeadbandRoom

### Test 1: Concurrent Agent Protocol Collision
**Scenario:** Multiple agents attempt P1 channel discovery simultaneously when only one safe channel exists.
**Edge Case:** Race condition in channel allocation.
**Test Purpose:** Verify protocol handles concurrent access without deadlock.

### Test 2: Tile Validation Under Memory Pressure
**Scenario:** Agent submits tiles while system memory is at 95% capacity.
**Edge Case:** Resource-constrained validation pipeline.
**Test Purpose:** Ensure tile validation completes without crashing under memory pressure.

### Test 3: Protocol Violation Recovery
**Scenario:** Agent attempts P2 optimization before completing P1 mapping.
**Edge Case:** Out-of-sequence protocol execution.
**Test Purpose:** Verify room detects violation and resets agent to appropriate protocol phase.

### Test 4: Malformed Training Tile Injection
**Scenario:** Agent submits tiles with corrupted metadata or invalid training data structure.
**Edge Case:** Non-compliant tile formats.
**Test Purpose:** Ensure room rejects malformed tiles without affecting other agents' sessions.

### Test 5: Boundary Condition: Maximum Concurrent Agents
**Scenario:** Room operates at exactly maximum agent capacity, then one agent disconnects and reconnects.
**Edge Case:** Capacity management at boundary limits.
**Test Purpose:** Verify room handles agent churn at capacity without protocol disruption.
