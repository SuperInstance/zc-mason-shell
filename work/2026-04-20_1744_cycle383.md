### Test 1: Consecutive P0 Calls with Identical Input
**Scenario:** Agent calls `map_negative_space()` twice with the same obstacle data.
**Expected:** Room should handle idempotency—either ignore duplicate or update timestamp without state corruption.
**Rationale:** Agents might retry or have redundant perception loops.

```python
def test_p0_idempotent_duplicate_obstacles(self):
    """Test that consecutive P0 calls with identical data don't corrupt state."""
    room = DeadbandRoom()
    obstacles = ["rock", "wall", "pit"]
    
    # First call
    result1 = room.map_negative_space(obstacles)
    self.assertEqual(room.current_phase, "P0")
    self.assertEqual(room.negative_space, obstacles)
    
    # Second identical call
    result2 = room.map_negative_space(obstacles)
    self.assertEqual(room.current_phase, "P0")  # Should still be P0
    self.assertEqual(room.negative_space, obstacles)  # State unchanged
```

### Test 5: Phase Rollback After Violation
**Scenario:** Agent violates protocol (skips to P2), gets error, then correctly returns to P1.
**Expected:** Room state should allow recovery without requiring full reset.
**Rationale:** Agents learn from mistakes; room should support correction.

```python
def test_phase_rollback_after_violation(self):
    """Test that room allows phase correction after protocol violation."""
    room = DeadbandRoom()
    room.map_negative_space(["rock"])
    
    # Attempt to skip to P2 (violation)
    with self.assertRaises(ProtocolError):
        room.optimize_within_channels()
    
    # Room should still be at P0 or P1, not locked
    self.assertIn(room.current_phase, ["P0", "P1"])
    
    # Now correctly do P1
    channels = room.find_safe_channels()
    self.assertEqual(room.current_phase, "P1")
    self.assertIsInstance(channels, list)
    
    # Then P2 should work
    room.optimize_within_channels()
    self.assertEqual(room.current_phase, "P2")
```
