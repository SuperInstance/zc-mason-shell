## 5 Edge-Case Tests for DeadbandRoom

### Test 1: Empty Negative Space (P0)
**Scenario:** Agent attempts to proceed when negative space is empty (no P0 constraints defined)
**Expected:** Room should raise specific error or default to "proceed with caution" state
**Rationale:** Real agents may encounter undefined environments

### Test 2: Overlapping Safe Channels (P1)
**Scenario:** Multiple safe channels intersect or conflict
**Expected:** Room should apply conflict resolution (priority, merging, or warning)
**Rationale:** Real-world constraints often create ambiguous safe zones

### Test 3: P2 Optimization with Zero-Gradient
**Scenario:** Safe channel identified but optimization gradient is flat (no clear "best" path)
**Expected:** Room should handle plateau detection and either random walk or status quo
**Rationale:** Optimization algorithms can stall on flat surfaces

### Test 4: Protocol Violation Chain (P0→P2 Direct)
**Scenario:** Agent correctly does P0, then incorrectly jumps to P2 without P1
**Expected:** Room should detect the P1 omission specifically, not just "skip to P2"
**Rationale:** Different violation types may require different corrective feedback

### Test 5: Tile Submission with Malformed Priority
**Scenario:** Agent submits tile with priority "P3" or "P-1" (outside 0-2 range)
**Expected:** Room should reject tile with specific error about priority range
**Rationale:** Input validation prevents corrupted training data
