## Edge-Case Tests Written

### Test 1: Protocol Priority Violation (P2 before P1)
**Scenario:** Agent submits a tile with P2 optimization content without demonstrating P1 safe channel identification.
**Expected:** Tile should be rejected or scored zero; room should log a protocol violation.
**Rationale:** Tests strict enforcement of Deadband Protocol's priority sequence.

### Test 2: Maximum Tile Size Boundary
**Scenario:** Agent submits a tile at exact maximum allowed size (e.g., 10,000 characters).
**Expected:** Tile should be accepted and processed normally.
**Rationale:** Tests buffer handling and input validation limits.

### Test 3: Concurrent Tile Submission Simulation
**Scenario:** Simulate two agents submitting tiles to the same room simultaneously.
**Expected:** Room should handle concurrent writes without data corruption or race conditions.
**Rationale:** Tests thread safety in real fleet multi-agent environments.

### Test 4: Malformed JSON in Tile Metadata
**Scenario:** Agent submits a tile with valid content but malformed JSON in metadata field.
**Expected:** Tile should be rejected with specific parsing error; room should remain stable.
**Rationale:** Tests robustness against malformed structured data.

### Test 5: Rapid Successive Protocol Phase Transitions
**Scenario:** Agent submits P0, P1, and P2 tiles in rapid succession (under 1 second intervals).
**Expected:** Room should process all tiles, maintain correct phase state tracking, and apply appropriate scoring.
**Rationale:** Tests state machine resilience under high-frequency input.
