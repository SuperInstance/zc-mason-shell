## 5 Additional Edge-Case Tests

### Test 1: Concurrent Agent State Collision
**Scenario:** Two agents attempt simultaneous P0→P1 transitions with conflicting negative space maps.  
**Edge:** Race condition in shared room state.  
**Expected:** Room should serialize transitions or detect conflicts, preventing invalid state.  
**Test Focus:** Thread safety and conflict resolution in `_apply_protocol_step()`.

### Test 2: Malformed Tile Payload Injection  
**Scenario:** Agent submits a tile with nested circular references or extreme depth.  
**Edge:** JSON serialization/validation boundaries.  
**Expected:** Room should reject tile with clear error, not crash.  
**Test Focus:** `submit_tile()` validation and `_validate_tile_structure()` robustness.

### Test 3: Protocol Priority Violation Recovery  
**Scenario:** Agent sends P2 optimization before completing P1 channel mapping.  
**Edge:** Out-of-order protocol steps.  
**Expected:** Room should reject P2, return agent to last valid P1 state, log violation.  
**Test Focus:** `_enforce_protocol_priority()` state machine integrity.

### Test 4: Ghost Tile Integration Under Load  
**Scenario:** Room contains 100+ ghost tiles from terminated agents, new agent joins.  
**Edge:** Memory/performance scaling of P0 knowledge base.  
**Expected:** Room should efficiently serve aggregated P0 constraints without degradation.  
**Test Focus:** `_load_ghost_tiles()` caching and query performance.

### Test 5: Network Partition Simulation  
**Scenario:** Mock agent session disconnects mid-transition, reconnects with stale state.  
**Edge:** Session recovery and state reconciliation.  
**Expected:** Room should detect stale session, reset or sync agent to current room state.  
**Test Focus:** `AgentSession` heartbeat timeout and `_reconcile_agent_state()`.

## Implementation Notes
These tests target the DeadbandRoom's resilience at scale and under stress—key for fleet training stability. Each test should:
1. Use existing pytest fixtures (`test_room`, `mock_agent`)
2. Isolate the specific edge condition
3. Verify both the failure mode and room's recovery
4. Maintain the existing test style (assertions on room state, logs, or returned errors)
