## Identified Gaps (Edge Cases)
The existing tests cover basic functionality but miss several edge cases:
1. **Extreme deadband widths** (e.g., very large values that exceed typical input ranges).
2. **NaN and infinity inputs** (floating-point edge cases).
3. **Input sequences** (time-series behavior, not just single values).
4. **Deadband width changes during operation** (dynamic reconfiguration).
5. **Mixed data types** (e.g., integer inputs with float deadband).
