## 5 Additional Edge-Case Tests

### Test 1: Empty Constraint Lists
**Scenario:** Agent attempts to operate with empty P0, P1, or P2 constraint lists.
**Expected Behavior:** Room should handle gracefully—possibly allowing progression if constraints are empty (no restrictions), but should not crash.
**Test Purpose:** Validate room resilience against malformed or minimal constraint configurations.

### Test 2: Overlapping Constraints Between Phases
**Scenario:** P0 constraint "avoid_rock" also appears in P1 or P2 lists.
**Expected Behavior:** Protocol violation? Ambiguity in phase priority. Room should detect and reject or log conflict.
**Test Purpose:** Ensure constraint uniqueness per phase or define clear precedence.

### Test 3: Agent Attempts P2 Before Completing P1 Actions
**Scenario:** Agent submits P2 optimization action but has not completed all required P1 safe-channel findings.
**Expected Behavior:** Room should block P2 action and return error state, enforcing strict phase ordering.
**Test Purpose:** Verify strict P0→P1→P2 sequencing beyond just P0→P2 skip.

### Test 4: Constraint with Special Characters or Very Long Strings
**Scenario:** Constraint strings contain newlines, emojis, or exceed typical length (e.g., 10k characters).
**Expected Behavior:** Room should sanitize, truncate, or reject based on implementation limits.
**Test Purpose:** Test input sanitization and robustness against injection or memory issues.

### Test 5: Concurrent Agent Interactions
**Scenario:** Multiple agents in same room attempt phase transitions simultaneously.
**Expected Behavior:** Room state should remain consistent; no race conditions causing invalid phase progress.
**Test Purpose:** Identify need for locking or transaction logic in room state management.
