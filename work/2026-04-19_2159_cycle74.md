## Edge-Case Tests Written

I have written 5 edge-case tests targeting specific vulnerabilities in the DeadbandRoom implementation:

### Test 1: `test_p0_rejection_with_p2_content`
**Scenario:** Agent submits a tile that contains P2 optimization content without completing P0 mapping
**Expected:** Tile rejected with "P0_NOT_COMPLETE" error
**Purpose:** Enforce strict priority protocol

### Test 2: `test_concurrent_agent_state_collision`
**Scenario:** Multiple agents attempt to update the same room state simultaneously
**Expected:** State locking mechanism prevents corruption, last write wins with conflict log
**Purpose:** Test thread safety in multi-agent environments

### Test 3: `test_malformed_tile_json_injection`
**Scenario:** Agent submits tile with malformed JSON containing injection attempts
**Expected:** Validation layer rejects tile before processing, logs security event
**Purpose:** Test input sanitization and security boundaries

### Test 4: `test_score_overflow_wraparound`
**Scenario:** Agent achieves maximum possible score (2^31-1) and submits another successful tile
**Expected:** Score counter handles overflow gracefully (saturates or uses bigint)
**Purpose:** Test numerical boundary conditions in scoring system

### Test 5: `test_partial_p1_with_insufficient_channels`
**Scenario:** Agent completes P0 but identifies only 1 safe channel (minimum threshold is 2)
**Expected:** Room remains in P1 phase, agent receives "INSUFFICIENT_CHANNELS" guidance
**Purpose:** Test minimum viability thresholds in protocol phases
