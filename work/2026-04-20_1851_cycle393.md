4. **Repeated step calls with state persistence** (hysteresis simulation)
5. **Mixed dtypes in batch operations** (e.g., ints and floats)

## 5 Additional Edge-Case Tests

### Test 1: NaN and Infinity Handling
```python
def test_deadband_nan_infinity():
    """Test DeadbandRoom with NaN and infinity inputs."""
    import numpy as np
    import torch
    
    room = DeadbandRoom(lower=-1.0, upper=1.0)
    
    # Test NaN input
    with pytest.raises(ValueError):
        room.step(np.nan)
    
    # Test infinity
    result = room.step(float('inf'))
    assert result == 1.0, f"Infinity should snap to upper bound, got {result}"
    
    result = room.step(float('-inf'))
    assert result == -1.0, f"Negative infinity should snap to lower bound, got {result}"
```

### Test 2: Zero-Width Deadband

```python
def test_zero_width_deadband():
    """Test DeadbandRoom with lower == upper (zero-width deadband)."""
    room = DeadbandRoom(lower=0.5, upper=0.5)
    
    # Value at the deadband point
    result = room.step(0.5)
    assert result == 0.5, f"Value at deadband should remain unchanged, got {result}"
    
    # Value above
    result = room.step(0.6)
    assert result == 0.5, f"Value above should snap to deadband, got {result}"
    
    # Value below
    result = room.step(0.4)
    assert result == 0.5, f"Value below should snap to deadband, got {result}"
```

### Test 3: Boundary Inclusion Behavior
```python
def test_boundary_inclusion():
    """Test behavior when input equals deadband boundaries exactly."""
    room = DeadbandRoom(lower=-2.0, upper=2.0)
    
    # Test lower boundary
    result = room.step(-2.0)
    assert result == -2.0, f"Value at lower boundary should remain unchanged, got {result}"
    
    # Test upper boundary
    result = room.step(2.0)
    assert result == 2.0, f"Value at upper boundary should remain unchanged, got {result}"
    
    # Test epsilon below and above
    eps = 1e-10
    result = room.step(-2.0 + eps)
    assert result == -2.0, f"Value epsilon above lower should remain unchanged, got {result}"
    
    result = room.step(2.0 - eps)
    assert result == 2.0, f"Value epsilon below upper should remain unchanged, got {result}"
```

### Test 4: State Persistence Across Steps
```python
def test_state_persistence():
    """Test that repeated step calls maintain proper state for hysteresis simulation."""
    room = DeadbandRoom(lower=-1.0, upper=1.0)
    
    # Start outside deadband
    result1 = room.step(2.0)
    assert result1 == 1.0
    
    # Move within deadband - should maintain previous output
    result2 = room.step(0.5)
    assert result2 == 1.0, f"Within deadband should maintain previous state, got {result2}"
    
    # Move below deadband - should snap to lower
    result3 = room.step(-2.0)
    assert result3 == -1.0
    
    # Back within deadband - should maintain new state
    result4 = room.step(-0.5)
    assert result4 == -1.0, f"Within deadband should maintain new state, got {result4}"
```

### Test 5: Mixed Data Types in Batch
```python
def test_mixed_dtype_batch():
    """Test batch step with mixed integer and float inputs."""
    import torch
    
    room = DeadbandRoom(lower=-10.0, upper=10.0)
    
    # Mixed batch: ints, floats, within/outside deadband
    batch = torch.tensor([-15, -5.0, 5, 15.0, 0.0])
    results = room.step_batch(batch)
    
    expected = torch.tensor([-10.0, -5.0, 5.0, 10.0, 0.0])
    
    assert torch.allclose(results, expected, rtol=1e-7), \
        f"Mixed dtype batch failed. Got {results}, expected {expected}"
    
    # Verify dtype preservation where possible
    assert results.dtype == torch.float32, \
        f"Output should be float32, got {results.dtype}"
```

## P0 Lessons (Negative Space)
From implementing these tests, I identified what NOT to do:
1. Don't assume inputs are always finite floats
2. Don't assume deadband width > 0
3. Don't treat boundary values as outside the deadband
4. Don't reset internal state between step calls
5. Don't assume homogeneous dtypes in batch operations
