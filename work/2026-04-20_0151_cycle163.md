## 5 Additional Edge-Case Tests

### 1. Test: `test_network_failure_during_tile_submission`
**Scenario:** Network connection drops when submitting a training tile.
**Expected:** Room should handle ConnectionError gracefully, log the failure, and retry according to retry policy without crashing.
**Edge:** Transient network failures should not corrupt room state.

### 2. Test: `test_malformed_tile_data_validation`
**Scenario:** Submit a tile with invalid JSON structure or missing required fields (e.g., no `agent_id`, `cycle`, `content`).
**Expected:** Room validation should reject the tile with a clear error message, not attempt to process it.
**Edge:** Malformed data from compromised or buggy agents should be safely discarded.

### 3. Test: `test_concurrent_tile_submissions`
**Scenario:** Multiple agents (threads) attempt to submit tiles simultaneously.
**Expected:** Room should handle concurrent submissions without race conditions (e.g., using thread locks or async queues). Tile counts should be accurate.
**Edge:** High-load scenarios where many agents submit simultaneously.

### 4. Test: `test_p0_empty_negative_space`
**Scenario:** Agent submits a P0 report with empty `negative_space` (no constraints learned).
**Expected:** Room should accept it as valid (agents may legitimately have no negatives to report) but log a warning if this persists across cycles.
**Edge:** Distinguishing between "no constraints" and "failed to analyze".

### 5. Test: `test_room_recovery_from_corrupted_state`
**Scenario:** Room's internal state file (`deadband_state.json`) becomes corrupted (invalid JSON, missing keys).
**Expected:** Room should detect corruption on load, reset to default initial state, and log the incident. Should not crash.
**Edge:** Filesystem or serialization errors should not permanently break the room.

## Implementation Notes
These tests would require:
- Mocking `requests.post` to simulate network failures
- Using `pytest.mark.parametrize` for malformed data variants
- `threading` or `asyncio` for concurrency tests
- Temporary files for state corruption tests
- Checking log outputs using `caplog` fixture
