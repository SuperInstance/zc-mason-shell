## Edge-Case Test Suite

### Test 1: P0 Violation with Valid P2 Input
**Scenario:** Agent attempts to optimize (P2) before establishing safe channels (P1)
**Expected:** Room should reject P2 input and return P0 violation error
**Purpose:** Tests strict priority enforcement when input is technically valid but out-of-sequence

```rust
#[test]
fn test_p2_input_before_p1_establishes_safe_channels() {
    let mut room = DeadbandRoom::new();
    
    // Agent tries to optimize (P2) without first establishing safe channels (P1)
    let input = "P2: optimize path with cost function";
    let response = room.process_input(input);
    
    assert!(response.contains("P0 violation"));
    assert!(response.contains("cannot optimize before safe channels"));
    assert_eq!(room.current_state(), RoomState::P0Mapping);
}
```

### Test 2: Empty P1 Channel Declaration
**Scenario:** Agent declares P1 completion with empty safe channels list
**Expected:** Room should reject empty channel declaration and remain in P1
**Purpose:** Tests validation of P1 channel completeness

```rust
#[test]
fn test_empty_p1_channel_declaration() {
    let mut room = DeadbandRoom::new();
    
    // Complete P0 mapping
    room.process_input("P0: avoid rocks, avoid collisions");
    
    // Attempt P1 with empty channels
    let input = "P1: channels: []";
    let response = room.process_input(input);
    
    assert!(response.contains("invalid P1 declaration"));
    assert!(response.contains("at least one safe channel required"));
    assert_eq!(room.current_state(), RoomState::P1ChannelFinding);
}
```

### Test 3: Maximum Tile Accumulation Boundary
**Scenario:** Room accumulates maximum allowed tiles (based on buffer size)
**Expected:** Room should stop accepting new tiles when buffer is full and return appropriate message
**Purpose:** Tests memory boundary conditions and tile buffer management

```rust
#[test]
fn test_max_tile_accumulation_boundary() {
    let mut room = DeadbandRoom::new();
    let max_tiles = room.get_max_tile_capacity();
    
    // Fill tile buffer to capacity
    for i in 0..max_tiles {
        room.process_input(&format!("P0: test tile {}", i));
        room.complete_current_priority();
    }
    
    // Attempt to add one more tile
    let response = room.process_input("P0: overflow tile");
    
    assert!(response.contains("tile buffer full"));
    assert!(response.contains("export required"));
    assert_eq!(room.get_tile_count(), max_tiles);
}
```

### Test 4: Concurrent Priority State Corruption
**Scenario:** Rapid state transitions with malformed input between valid transitions
**Expected:** Room should maintain consistent state and reject malformed transitions
**Purpose:** Tests state machine resilience under rapid, mixed input

```rust
#[test]
fn test_concurrent_priority_state_corruption() {
    let mut room = DeadbandRoom::new();
    
    // Valid P0 completion
    room.process_input("P0: map negative space");
    room.complete_current_priority();
    
    // Attempt invalid P2 while in P1
    let invalid_response = room.process_input("P2: premature optimization");
    assert!(invalid_response.contains("P1 required first"));
    
    // Valid P1 declaration
    let valid_response = room.process_input("P1: channels: [navigation, communication]");
    assert!(valid_response.contains("safe channels established"));
    
    // State should be consistent
    assert_eq!(room.current_state(), RoomState::P2OptimizationReady);
}
```

### Test 5: Unicode and Special Character Input Handling
**Scenario:** Agent uses Unicode characters, emojis, and special symbols in protocol declarations
**Expected:** Room should parse valid protocol commands regardless of special characters, but reject if they break protocol syntax
**Purpose:** Tests input sanitization and protocol parsing robustness

```rust
#[test]
fn test_unicode_and_special_character_handling() {
    let mut room = DeadbandRoom::new();
    
    // P0 with Unicode and special chars (should parse)
    let input_unicode = "P0: avoid ⚡ rocks 🪨, don't collide 🚫";
    let response = room.process_input(input_unicode);
    assert!(response.contains("P0 mapping accepted"));
    
    // Invalid protocol with broken syntax due to special chars
    let input_broken = "P1: channels: [nav🚫, comm📡]";
    let response_broken = room.process_input(input_broken);
    assert!(response_broken.contains("parse error"));
    
    // Valid P1 with allowed special chars
    let input_valid = "P1: channels: [nav-gps, comm-radio]";
    let response_valid = room.process_input(input_valid);
    assert!(response_valid.contains("safe channels"));
}
```
