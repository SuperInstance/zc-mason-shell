## New Edge-Case Tests

Based on the code, I identified five edge cases not covered by existing tests:

### 1. **Test: P1 Phase with Empty Negative Space**
**Scenario:** Agent completes P0 (mapping negative space) but provides an empty list. According to protocol, P1 should still execute to find safe channels.
```python
def test_p1_with_empty_negative_space(mock_agent, sample_tile):
    """Edge case: P0 returns empty negative_space, P1 should still run."""
    room = DeadbandRoom()
    room.register_agent(mock_agent)
    
    # Agent submits P0 tile with empty negative_space
    tile = sample_tile.copy()
    tile["phase"] = "P0"
    tile["negative_space"] = []  # Empty list
    room.submit_tile(mock_agent.agent_id, tile)
    
    assert room.current_phase == "P1"
    assert room.negative_space == []
    # Room should still accept P1 tiles
    assert room.can_accept_tile(mock_agent.agent_id, "P1") is True
```

### 2. **Test: Concurrent Agent Registration During Active Phase**
**Scenario:** New agent tries to register while room is in P1 phase. Should be allowed but start at current phase.
```python
def test_late_agent_registration(mock_agent):
    """Edge case: Agent registers after room has advanced to P1."""
    room = DeadbandRoom()
    room.register_agent(mock_agent)
    
    # Advance room to P1
    room.current_phase = "P1"
    room.negative_space = ["rock1", "rock2"]
    
    # New agent tries to register
    new_agent = Mock(spec=Agent, agent_id="agent2")
    room.register_agent(new_agent)
    
    assert new_agent.agent_id in room.registered_agents
    # New agent should start at current phase (P1)
    assert room.get_agent_phase(new_agent.agent_id) == "P1"
    # Should not be able to submit P0 tiles
    assert room.can_accept_tile(new_agent.agent_id, "P0") is False
```

### 4. **Test: P2 Optimization with Single Safe Channel**
**Scenario:** P1 identifies only one safe channel. P2 should still attempt to optimize within it (not skip).
