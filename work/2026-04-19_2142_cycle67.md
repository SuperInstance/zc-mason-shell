## Proposed Edge-Case Tests for DeadbandRoom

**Test 1: P0 Violation Recovery**
- **Scenario:** Agent submits a tile that explicitly violates a known P0 constraint (e.g., attempts to optimize before mapping negative space).
- **Expected Behavior:** Room rejects the tile, logs a P0 violation, and provides a corrective feedback tile.
- **Edge Case:** The violation is subtle—agent skips P1 but claims to be in P2.

**Test 2: Empty Safe Channel (P1)**
- **Scenario:** Agent correctly identifies no safe channels (all paths blocked by P0 constraints).
- **Expected Behavior:** Room enters a "deadlock" state, generates a diagnostic tile suggesting re-evaluation of P0 mapping, and does not advance to P2.
- **Edge Case:** Distinguishing between "no channels" and "channels not yet found".

**Test 3: High-Velocity Tile Flood**
- **Scenario:** Agent submits 100+ training tiles in rapid succession (simulating a burst workload).
- **Expected Behavior:** Room queues tiles, processes them in priority order (P0 > P1 > P2), and throttles if necessary to prevent resource exhaustion.
- **Edge Case:** Ensuring tile integrity and order are preserved under load.

**Test 4: Malformed Tile Payload**
- **Scenario:** Agent sends a tile with missing required fields (e.g., no phase indicator, invalid JSON structure).
- **Expected Behavior:** Room rejects the tile with a structured error, increments a malformed-tile counter, and does not affect training state.
- **Edge Case:** Partially valid tiles that pass schema but violate semantic rules.

**Test 5: Protocol Loop Detection**
- **Scenario:** Agent repeatedly oscillates between P1 and P2 without making progress (e.g., finds a channel, optimizes, then re-evaluates and abandons it).
- **Expected Behavior:** Room detects cyclical behavior (e.g., via state fingerprinting), halts further training, and generates an intervention tile requiring manual review.
- **Edge Case:** Differentiating between legitimate iteration and a stuck loop.

## Rationale
These tests target:
- **Doctrine adherence** (P0 > P1 > P2 strict priority)
- **Robustness** under abnormal inputs
- **State consistency** across edge conditions
- **Recovery mechanisms** when agents deviate
