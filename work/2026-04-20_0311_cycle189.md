## Analysis of Existing Test Suite
The existing `test_deadband.py` contains 12 tests covering:
- Basic room initialization (`test_room_initialization`)

The tests use pytest fixtures and mock the underlying PLATO room server. Edge cases around network failures, malformed data, and protocol violations are not fully covered.

## 5 Additional Edge-Case Tests

### 1. Test: `test_p0_step_with_empty_negative_space`
**Scenario:** Agent attempts P0 mapping when the negative space list is empty (no prior P0 knowledge).
**Expected Behavior:** Room should return safe default channel or raise informative error, not crash.
**Rationale:** Ensures robustness when initializing a new room without historical P0 data.

### 2. Test: `test_submit_tile_with_malformed_json`
**Scenario:** Agent submits a tile where the `content` field contains invalid JSON structure.
**Expected Behavior:** Room should reject the tile with a `400` error and clear error message, not propagate JSON decoding exceptions.
**Rationale:** Validates input sanitization and graceful error handling.

### 3. Test: `test_concurrent_agent_state_conflict`
**Scenario:** Two agents attempt to transition to P2 simultaneously when only one "safe channel" exists.
**Expected Behavior:** Room should handle the race condition via locking or conflict resolution, preventing both agents from occupying the same channel.
**Rationale:** Tests thread safety and protocol enforcement under concurrency.

### 4. Test: `test_agent_reconnects_with_stale_state`
**Scenario:** Agent disconnects mid-P1, reconnects later with outdated session state.
**Expected Behavior:** Room should detect stale state, reset the agent to a valid prior step (P0), or provide a recovery mechanism.
**Rationale:** Ensures resilience to network interruptions and state synchronization.

### 5. Test: `test_protocol_skip_from_p0_to_p2`
**Scenario:** Agent maliciously or erroneously attempts to jump from P0 (negative space) directly to P2 (optimization), bypassing P1.
**Expected Behavior:** Room should reject the transition, log a protocol violation, and keep the agent at P0.
**Rationale:** Enforces strict priority of Deadband Protocol (P0→P1→P2) as defined in fleet doctrine.
