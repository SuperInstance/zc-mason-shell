## 5 Edge-Case Tests for DeadbandRoom

### Test 1: Protocol Priority Violation Recovery
**Scenario:** Agent attempts P2 optimization while P0 negative space is unmapped.
**Expected:** Room rejects tile, logs P0 violation, returns agent to P0 mapping state.
**Edge:** Ensures strict priority is enforced even if agent tries to "skip ahead."

### Test 2: Empty Tile Submission
**Scenario:** Agent submits a tile with zero content (empty string or null payload).
**Expected:** Room marks tile as "invalid," does not increment training count, provides error feedback.
**Edge:** Validates room resilience to malformed inputs without crashing.

### Test 3: Concurrent Tile Submission from Multiple Agents
**Scenario:** Two agents submit tiles simultaneously for same room instance.
**Expected:** Room processes tiles sequentially (or with locking), maintains consistent state, no data corruption.
**Edge:** Tests thread safety and state integrity under load.

### Test 4: P1 Channel Exhaustion
**Scenario:** Agent exhausts all known safe channels (P1), no viable paths remain.
**Expected:** Room transitions agent back to P0 to map new negative space, does not allow stagnation.
**Edge:** Ensures protocol loops correctly when channels are fully explored.

### Test 5: Ghost Tile Integration
**Scenario:** Room loads with existing ghost tiles (lessons from dead agents) in memory.
**Expected:** Room incorporates ghost tiles as P0 knowledge, new agents see them as constraints.
**Edge:** Validates persistence and integration of historical failure data.

## Implementation Notes
- Tests should be written in pytest format (consistent with plato-torch).
- Mock external dependencies (e.g., tile server, agent runtime).
- Assert both state changes and log outputs.
- Focus on observable behavior, not internal implementation.
