## Analysis of Existing Tests
From `test_deadband_room.py`, I observe the following test patterns:
1. **Basic Protocol Flow**: Tests P0→P1→P2 progression with valid inputs.
2. **Negative Space Detection**: Tests that invalid actions (P0 violations) are caught.
3. **Channel Validation**: Tests that P1 channels are correctly identified.
4. **Optimization Bounds**: Tests P2 optimization respects channel constraints.
5. **State Persistence**: Tests that room state persists across interactions.

The existing tests cover happy paths and basic error cases. Edge cases involve boundary conditions, malformed inputs, concurrent interactions, and protocol violation sequences.

## Proposed Edge-Case Tests

### Test 1: Empty Action Sequence
**Scenario**: Agent submits an empty list of actions for P0 mapping.
**Edge**: The protocol expects at least one action to evaluate. Should handle empty input gracefully without crashing.
**Assertion**: Room returns a clear error state (e.g., `"no_actions"`) rather than proceeding.

### Test 2: Maximum Channel Saturation
**Scenario**: Agent identifies all possible P1 channels as "safe," saturating the channel buffer.
**Edge**: Tests the upper limit of channel storage. Does the room truncate, reject, or overflow?
**Assertion**: Room accepts channels up to its defined limit (e.g., 100) and rejects or truncates beyond that.

### Test 3: P2 Optimization with Zero Valid Channels
**Scenario**: After P1, zero safe channels are found (all are negative space).
**Edge**: Protocol cannot proceed to P2. Should it loop back to P0, halt, or error?
**Assertion**: Room enters a "deadlock" state and requires a reset or new P0 input.

### Test 4: Malformed JSON in Tile Payload
**Scenario**: Agent submits a tile with valid metadata but a malformed JSON string in the `content` field.
**Edge**: The room's parser must handle JSON decode errors without corrupting state.
**Assertion**: Tile is rejected with a `"parse_error"` flag, and room state remains unchanged.

### Test 5: Concurrent Tile Submission Simulation
**Scenario**: Simulate two agents submitting tiles to the same room within a short time window.
**Edge**: Tests state mutation under concurrency. Does the room use locking, queues, or last-write-wins?
**Assertion**: Room processes tiles sequentially (e.g., via a queue) and state reflects the combined sequence correctly.
