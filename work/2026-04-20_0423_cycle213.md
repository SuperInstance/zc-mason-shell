## 5 Additional Edge-Case Tests

### Test 1: Empty Action Space (No Safe Channels)
**Scenario:** When P0 constraints eliminate all possible actions, verify the room correctly identifies no P1 safe channels and raises appropriate error.

```python
def test_empty_action_space_no_p1_channels():
    """Test DeadbandRoom when P0 constraints eliminate all actions."""
    room = DeadbandRoom(
        action_bounds=[(-10, 10)],
        p0_constraints=[lambda x: x < -20],  # Impossible constraint
        p1_channels=[lambda x: -5 <= x <= 5],
        p2_optimizer=lambda x: x * 2
    )
    
    with pytest.raises(ValueError, match="No valid P1 channels found"):
        room.process_action(0)
```

### Test 2: Overlapping P0 and P1 Boundaries
**Scenario:** When P0 constraints and P1 channels have overlapping boundaries at exact values, verify priority enforcement (P0 wins).

```python
def test_overlapping_p0_p1_boundaries():
    """Test boundary case where P0 and P1 definitions overlap."""
    room = DeadbandRoom(
        action_bounds=[(0, 100)],
        p0_constraints=[lambda x: x >= 50],  # P0: x >= 50 is forbidden
        p1_channels=[lambda x: 40 <= x <= 60],  # P1: safe between 40-60
        p2_optimizer=lambda x: x + 10
    )
    
    # Action at exact boundary 50 should fail P0
    result = room.process_action(50)
    assert not result["p0_valid"]
    assert result["p1_valid"] is False  # P1 not evaluated due to P0 failure
    
    # Action at 49 should pass P0, potentially pass P1
    result = room.process_action(49)
    assert result["p0_valid"]
    assert result["p1_valid"]  # 49 is in [40, 60]
```

### Test 3: Discontinuous P1 Channels
**Scenario:** Test with non-contiguous safe channels (multiple disjoint safe ranges).

```python
def test_discontinuous_p1_channels():
    """Test DeadbandRoom with multiple disjoint safe channels."""
    room = DeadbandRoom(
        action_bounds=[(-100, 100)],
        p0_constraints=[lambda x: -10 <= x <= 10],  # Forbid [-10, 10]
        p1_channels=[
            lambda x: -50 <= x <= -20,
            lambda x: 20 <= x <= 50
        ],  # Two separate safe zones
        p2_optimizer=lambda x: abs(x)
    )
    
    # Test in first safe channel
    result = room.process_action(-35)
    assert result["p0_valid"]
    assert result["p1_valid"]
    assert result["p2_optimized"] == 35
    
    # Test in second safe channel  
    result = room.process_action(30)
    assert result["p0_valid"]
    assert result["p1_valid"]
    assert result["p2_optimized"] == 30
    
    # Test in forbidden gap between channels
    result = room.process_action(15)
    assert result["p0_valid"]
    assert not result["p1_valid"]  # Not in any safe channel
```

### Test 4: P2 Optimizer Returns Out-of-Bounds Value
**Scenario:** When P2 optimizer produces a value outside action bounds, verify proper clamping or error.

```python
def test_p2_optimizer_exceeds_bounds():
    """Test when P2 optimization produces value outside action bounds."""
    room = DeadbandRoom(
        action_bounds=[(0, 100)],
        p0_constraints=[lambda x: x < 0 or x > 100],
        p1_channels=[lambda x: 20 <= x <= 80],
        p2_optimizer=lambda x: x * 2  # Can exceed 100
    )
    
    # Action that when doubled exceeds upper bound
    result = room.process_action(60)
    assert result["p0_valid"]
    assert result["p1_valid"]
    
    # The room should handle out-of-bounds P2 output
    # Either clamp to bounds or raise error
    p2_value = result["p2_optimized"]
    assert 0 <= p2_value <= 100, f"P2 value {p2_value} should be clamped to bounds"
```

### Test 5: State Persistence Across Multiple Actions
**Scenario:** Verify room maintains consistent state when processing sequential actions.

```python
def test_state_persistence_sequential_actions():
    """Test DeadbandRoom state across multiple action processing calls."""
    room = DeadbandRoom(
        action_bounds=[(-1000, 1000)],
        p0_constraints=[lambda x: abs(x) < 100],  # Forbid small magnitudes
        p1_channels=[lambda x: 200 <= abs(x) <= 500],
        p2_optimizer=lambda x: x * 0.5  # Reduce magnitude
    )
    
    # First action
    result1 = room.process_action(400)
    assert result1["p0_valid"]
    assert result1["p1_valid"]
    assert result1["p2_optimized"] == 200
    
    # Second action - room should not be contaminated by previous state
    result2 = room.process_action(-450)
    assert result2["p0_valid"]
    assert result2["p1_valid"]
    assert result2["p2_optimized"] == -225
    
    # Third action that violates P0
    result3 = room.process_action(50)
    assert not result3["p0_valid"]
    assert result3["p1_valid"] is False
```

## Test Rationale
These edge cases address:
1. **Empty solution space** - Critical failure mode for deadband protocol
2. **Boundary collisions** - Real-world constraint systems often have overlapping boundaries
3. **Discontinuous safe zones** - Common in multi-modal optimization
4. **Optimizer boundary violations** - Optimizers can overshoot
5. **State isolation** - Ensures no cross-contamination between evaluations
