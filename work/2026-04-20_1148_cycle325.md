## Analysis of Existing Tests
The `DeadbandRoom` class in `plato-torch/rooms/deadband_room.py` implements the fleet's Deadband Protocol (P0, P1, P2). Existing tests in `test_deadband_room.py` cover:

Observed patterns:
- Tests use `pytest` and `unittest.mock`
- `DeadbandRoom` is instantiated with a `room_id` and optional `config`
- Core method: `process_tile(tile_data)` returns a dict with `status`, `message`, `next_priority`
- Tiles are dictionaries expected to contain `priority`, `content`, `agent_id`

## Edge-Case Test Proposals
Based on the code structure and protocol logic, I identified the following edge cases not covered in existing tests:

### 1. **Test: P0 Tile with Empty Negative Space Map**
**Scenario:** A tile submitted at P0 with `content` that is an empty list or dict for "negative space."
**Expected Behavior:** The room should accept it (P0 is about mapping, even if empty) and transition to P1.
**Rationale:** Ensures the room doesn't crash on empty P0 declarations.

### 2. **Test: Malformed Tile Missing Required Fields**
**Scenario:** A tile dictionary missing the `priority` field.
**Expected Behavior:** `process_tile` should return `{"status": "error", "message": "Missing required field: priority"}`.
**Rationale:** Input validation robustness.

### 3. **Test: Invalid Priority Transition (Skip from P0 to P2)**
**Scenario:** A tile submitted at P2 when the room's current state is P0.
**Expected Behavior:** The room should reject the tile with a message about protocol violation, state remains P0.
**Rationale:** Enforces strict P0→P1→P2 sequencing.

### 4. **Test: Large Tile Content (Boundary/Size Check)**
**Scenario:** A tile with `content` exceeding a plausible size (e.g., 10 MB string).
**Expected Behavior:** The room may reject it or truncate based on `config`; test should verify no crash.
**Rationale:** Stress test for memory/performance.

### 5. **Test: Concurrent Tile Submissions (Thread Safety)**
**Scenario:** Simulate two threads calling `process_tile` simultaneously.
**Expected Behavior:** State should remain consistent (e.g., no race condition in priority transition).
**Rationale:** The room may be used in async contexts; basic thread safety is desirable.

import pytest
import threading
import time

def test_p0_empty_negative_space():
    """Edge case: P0 tile with empty negative space map."""
    from plato_torch.rooms.deadband_room import DeadbandRoom
    room = DeadbandRoom(room_id="test_edge_1")
    tile = {
        "priority": "P0",
        "content": {"negative_space": []},
        "agent_id": "mason"
    }
    result = room.process_tile(tile)
    assert result["status"] == "accepted"
    assert result["next_priority"] == "P1"
    assert "Empty P0 map accepted" in result.get("message", "")

def test_malformed_tile_missing_priority():
    """Edge case: Tile missing required 'priority' field."""
    from plato_torch.rooms.deadband_room import DeadbandRoom
    room = DeadbandRoom(room_id="test_edge_2")
    tile = {
        "content": "some content",
        "agent_id": "mason"
    }
    result = room.process_tile(tile)
    assert result["status"] == "error"
    assert "Missing required field: priority" in result["message"]

def test_invalid_priority_transition_skip_p0_to_p2():
    """Edge case: Attempt to skip from P0 directly to P2."""
    from plato_torch.rooms.deadband_room import DeadbandRoom
    room = DeadbandRoom(room_id="test_edge_3")
    # First, submit a valid P0 tile to set state
    room.process_tile({"priority": "P0", "content": {}, "agent_id": "mason"})
    # Now try P2
    tile = {
        "priority": "P2",
        "content": {"optimization": "fast"},
        "agent_id": "mason"
    }
    result = room.process_tile(tile)
    assert result["status"] == "rejected"
    assert "Protocol violation" in result["message"]
    assert room.current_priority == "P0"  # State unchanged

def test_large_tile_content():
    """Edge case: Tile with excessively large content."""
    from plato_torch.rooms.deadband_room import DeadbandRoom
    room = DeadbandRoom(room_id="test_edge_4", config={"max_tile_size": 1024*1024})  # 1 MB
    large_content = "x" * (2 * 1024 * 1024)  # 2 MB string
    tile = {
        "priority": "P0",
        "content": {"data": large_content},
        "agent_id": "mason"
    }
    result = room.process_tile(tile)
    # Expect rejection or truncation per config
    assert result["status"] in ["rejected", "accepted"]
    if result["status"] == "rejected":
        assert "exceeds size limit" in result["message"].lower()

def test_concurrent_tile_submissions():
    """Edge case: Two threads submitting tiles simultaneously."""
    from plato_torch.rooms.deadband_room import DeadbandRoom
    room = DeadbandRoom(room_id="test_edge_5")
    results = []
    errors = []

    def submit_tile(priority):
        try:
            tile = {"priority": priority, "content": {}, "agent_id": "mason"}
            result = room.process_tile(tile)
            results.append((priority, result))
        except Exception as e:
            errors.append(e)

    # Start two threads: one P0, one P1 (should be serialized)
    t1 = threading.Thread(target=submit_tile, args=("P0",))
    t2 = threading.Thread(target=submit_tile, args=("P1",))
    t1.start()
    t2.start()
    t1.join()
    t2.join()

    # Check for errors (none expected)
    assert len(errors) == 0
    # At least one tile should be accepted
    accepted = [r for _, r in results if r.get("status") == "accepted"]
    assert len(accepted) >= 1
    # State should be consistent (e.g., final priority is P1 or P2)
    assert room.current_priority in ["P0", "P1", "P2"]
