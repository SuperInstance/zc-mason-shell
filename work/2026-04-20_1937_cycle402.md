## Proposed Edge-Case Tests
Based on the DeadbandRoom implementation and fleet doctrine, here are 5 additional edge-case tests:

### Test 1: Invalid Protocol Priority Skipping (P0 Violation)
**Scenario:** Agent attempts to execute P2 optimization without completing P1 channel identification.
**Expected Behavior:** Room should reject the tile with appropriate error, logging a P0 violation.
**Test Value:** Ensures strict adherence to "Never skip to P2" doctrine.

### Test 2: Malformed Tile JSON Structure
**Scenario:** Agent submits a tile with missing required fields or invalid JSON syntax.
**Expected Behavior:** Room should handle parsing errors gracefully, returning a validation error without crashing.
**Test Value:** Robustness against malformed client submissions.

### Test 3: Concurrent Tile Submission Race Condition
**Scenario:** Multiple agents attempt to submit tiles simultaneously for the same cycle.
**Expected Behavior:** Room should handle concurrent writes safely, either through locking or conflict resolution.
**Test Value:** Thread safety in multi-agent environments.

### Test 4: Extreme Input Values (Boundary Testing)
**Scenario:** Agent submits tiles with extremely large payloads, negative cycle counts, or empty content.
**Expected Behavior:** Room should validate input boundaries and reject unreasonable values.
**Test Value:** Protection against resource exhaustion and invalid state.

### Test 5: Room State Corruption Recovery
**Scenario:** Simulate corrupted room state file (e.g., invalid JSON, missing keys).
**Expected Behavior:** Room should either recover to a known good state or initialize fresh with appropriate logging.
**Test Value:** Resilience to storage layer failures.
