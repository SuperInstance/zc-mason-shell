Based on P0 and P1, here are 5 specific, implementable edge-case tests.

### Test 1: P0 Completion with Empty Negative Space Map
**Scenario:** An agent submits a P0 result that is a valid JSON object but contains an empty list or dict for `negative_space`.
**Expected Behavior:** The protocol should advance to P1 (finding safe channels), as an empty negative space is technically valid ("nothing is prohibited"). The test should verify state transition and appropriate log/tile message.
**Assertions:** `self.state['phase'] == 'P1'`, output tile contains P1 prompt.

### Test 2: P1 Submission with Overlapping/Contradictory Channels
**Scenario:** An agent submits a P1 result where `safe_channels` contains entries that logically contradict the established `negative_space` (e.g., a channel that is explicitly listed as prohibited in P0).
**Expected Behavior:** The room should reject the P1 submission, remain in P1, and send a tile indicating the contradiction. It should **not** advance to P2.
**Assertions:** `self.state['phase'] == 'P1'`, output tile indicates validation failure, `self.state['safe_channels']` is not updated.

### Test 3: Malformed Tile in Mid-Protocol (P1 or P2)
**Scenario:** The room is in phase P1. An agent sends a tile that is well-formed JSON but is structured as a P0 tile (or a P2 tile).
**Expected Behavior:** The room should reject the tile as "out-of-phase," send an error message, and maintain its current state and phase.
**Assertions:** `self.state['phase']` unchanged, `self.state['safe_channels']` unchanged, output tile contains phase error.

### Test 4: Direct P2 Submission After P0 (Skipping P1)
**Scenario:** An agent completes P0 and, in the next interaction, sends a tile formatted for P2, attempting to skip P1 entirely.
**Expected Behavior:** The room should reject the submission. The existing test checks for a jump *to* P2 logic; this test checks for a malformed submission that *is* P2 format. It should remain in P1.
**Assertions:** `self.state['phase'] == 'P1'`, output tile indicates P1 is required.

### Test 5: Reset or New Session Interruption
**Scenario:** Simulate a new agent joining an existing room session. The room has state from a previous agent (P2 optimized path). The new agent sends a simple "hello" or session-start tile.
**Expected Behavior:** The room's behavior depends on its design. Does it reset state for a new agent? Does it expect the new agent to continue the existing protocol? The test should document this behavior. The edge case is the interaction between persisted room state and a new user session.
**Assertions:** Will depend on implementation. Could assert state is reset (`self.state['phase'] == 'P0'`) or that a continuation prompt is sent.
