## Edge-Case Tests Written

I created `test_deadband_edge.py` with 5 new tests:

### 1. Test: `test_step_with_empty_deadband_rules`
- **Edge Case:** What happens when the deadband rules list is empty?
- **Rationale:** The protocol depends on P0 rules; empty rules could indicate misconfiguration.
- **Test:** Initializes room with `deadband_rules=[]`, calls `step()` with a valid action, checks that room still functions (doesn't crash) but may produce warnings or default behavior.

### 2. Test: `test_step_with_none_action`
- **Edge Case:** Passing `None` as action to `step()`.
- **Rationale:** `None` is a common Python sentinel; should be handled gracefully.
- **Test:** Calls `room.step(None)`, expects `ValueError` or specific handling per implementation.

### 3. Test: `test_reset_preserves_configuration`
- **Edge Case:** After `reset()`, does the room retain its original configuration (deadband_rules, max_steps)?
- **Rationale:** `reset()` should clear episodic state but not static configuration.
- **Test:** Stores original config, calls `reset()`, asserts config values unchanged.

### 4. Test: `test_step_exceeds_max_steps`
- **Edge Case:** What happens when `step()` is called beyond `max_steps`?
- **Rationale:** Rooms may have episode length limits; should handle gracefully (e.g., truncate, signal done).
- **Test:** Calls `step()` in a loop exceeding `max_steps`, checks for `done` flag or truncation.

### 5. Test: `test_deadband_rule_with_zero_threshold`
- **Edge Case:** A deadband rule with `threshold=0.0`.
- **Rationale:** Zero threshold means any violation triggers P0; tests sensitivity edge.
- **Test:** Creates rule with `threshold=0.0`, provides action that slightly violates, expects P0 rejection.
