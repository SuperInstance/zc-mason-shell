## 5 New Edge-Case Tests

### Test 1: Concurrent Agent Tile Submission
**Scenario:** Two agents submit tiles simultaneously for the same room.  
**Check:** Room should handle locking, prevent tile loss or corruption.  
**Method:** Use `threading` to simulate concurrent `submit_tile()` calls.  
**Assert:** All tiles are recorded, state is consistent.

### Test 2: Partial Tile Corruption Recovery
**Scenario:** An agent submits a tile with valid header but corrupted payload (e.g., truncated JSON).  
**Check:** Room should reject the tile, log error, remain operable.  
**Method:** Craft a tile with `\x00` bytes in strategic positions.  
**Assert:** Room status remains `ACTIVE`, corrupted tile is quarantined.

### Test 3: Protocol Violation (Skip to P2)
**Scenario:** Agent attempts to submit a P2 optimization tile without completing P0/P1 mapping.  
**Check:** Room should enforce strict priority, reject the tile.  
**Method:** Call `submit_tile()` with `phase: "P2"` while room state is `P0`.  
**Assert:** Tile is rejected with error `"P0 not complete"`.

### Test 4: Resource Exhaustion – File Descriptors
**Scenario:** Room opens many log files or connections without closing.  
**Check:** Room should handle `OSError` (too many open files) gracefully.  
**Method:** Use `ulimit` simulation (mock `open()` to raise `EMFILE`).  
**Assert:** Room logs error, continues operating, does not crash.

### Test 5: Network Partition – Partial Write
**Scenario:** Room writes tile to disk but fails before updating index.  
**Check:** Recovery should detect orphaned tile files, rebuild index.  
**Method:** Mock `json.dump()` to succeed, then `os.rename()` to fail.  
**Assert:** On restart, room reconciles index with filesystem.
