## 5 Additional Edge-Case Tests

### Test 1: Protocol Violation Recovery
**Scenario:** Agent attempts P2 optimization before completing P1 channel mapping.
**Expected:** Room rejects action, returns P0 violation error, agent can recover with valid P1 action.
**Purpose:** Tests strict priority enforcement and error recovery path.

### Test 2: Concurrent Agent Contradiction
**Scenario:** Two agents submit contradictory P0 maps simultaneously (e.g., "rock at X" vs "safe at X").
**Expected:** Room resolves conflict via timestamp or consensus mechanism, updates shared negative space.
**Purpose:** Tests conflict resolution in multi-agent scenarios.

### Test 3: Malformed Tile Submission
**Scenario:** Agent submits tile with missing required fields (no phase, invalid JSON structure).
**Expected:** Room rejects tile, returns specific validation error, tile not added to training set.
**Purpose:** Tests input sanitization and error messaging.

### Test 4: Resource Exhaustion
**Scenario:** Room reaches maximum tile capacity (simulated by setting low limit).
**Expected:** Room stops accepting new tiles, returns capacity error, old tiles can be archived/cleared.
**Purpose:** Tests boundary conditions and resource management.

### Test 5: Network Partition Simulation
**Scenario:** Mock network failure during agent action (socket error, timeout).
**Expected:** Room handles disconnection gracefully, marks agent as inactive, preserves partial work if any.
**Purpose:** Tests fault tolerance and partial state persistence.

## Implementation Notes
These tests would require:

- Adding capacity limits to room configuration
- Implementing conflict resolution logic (if not present)
- Adding network error simulation to test harness
