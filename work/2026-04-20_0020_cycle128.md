## Edge-Case Test Suite (5 Tests)

### Test 1: Protocol Priority Violation (Skipping to P2)
**Scenario:** Agent attempts to submit P2 optimization tiles without completing P1 safe channel mapping.
**Expected:** Room rejects tile with "P1_NOT_COMPLETE" error.
**Rationale:** Enforces strict priority doctrine (P0→P1→P2).

### Test 2: Negative Space Contradiction
**Scenario:** Agent submits P0 negative space tile that directly contradicts previously accepted P0 knowledge.
**Expected:** Room flags contradiction, requires human review, creates ghost tile if rejected.
**Rationale:** Ensures P0 knowledge base remains internally consistent.

### Test 3: Channel Boundary Testing
**Scenario:** Agent submits P1 safe channel tile with boundary conditions (e.g., channel width = 0, overlapping channels).
**Expected:** Room validates geometric constraints, rejects invalid channels, accepts valid edge cases.
**Rationale:** Tests constraint-theory-core integration for geometric validation.

### Test 4: Tile Flood Protection
**Scenario:** Agent submits 100+ tiles in rapid succession (simulating malfunction or attack).
**Expected:** Room implements rate limiting, queues excess tiles, maintains protocol state integrity.
**Rationale:** Ensures room stability under high load or adversarial conditions.

### Test 5: Protocol Reset Conditions
**Scenario:** After completing P2 optimization, agent requests protocol reset to P0 for new problem domain.
**Expected:** Room clears previous tiles, resets state machine, allows new P0 submissions.
**Rationale:** Tests room's ability to handle multiple sequential problem-solving sessions.
