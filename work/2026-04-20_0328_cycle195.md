def test_protocol_violation_recovery():
    """Test that room recovers when P2 is attempted before P1 completion."""
    room = DeadbandRoom()
    # Force invalid state where P2 optimization attempted prematurely
    room.current_state = {"p0_complete": True, "p1_complete": False}
    room.current_state["attempted_p2"] = True
    
    # Room should detect violation and reset to P1
    room.validate_state()
    assert room.current_state["p1_complete"] == False
    assert room.current_state["violation_count"] == 1
    assert "p2_optimization" not in room.current_state
```

### Test 4: Memory Exhaustion Scenario

def test_large_input_handling():
    """Test behavior with extremely large input that could exhaust memory."""
    room = DeadbandRoom()
    
    # 10MB string (simulating potential DoS)
    large_input = "x" * (10 * 1024 * 1024)
    
    # Room should reject or truncate before full processing
    result = room.process_input(large_input)
    
    # Acceptable outcomes:
    # 1. Rejected with error
    # 2. Truncated and processed partially
    # 3. Processed with performance warning
    
    assert result["status"] != "crash"
    if result["status"] == "error":
        assert "too large" in result["message"].lower() or "memory" in result["message"].lower()
    elif result["status"] == "success":
        assert len(room.current_input) < len(large_input)  # Was truncated
        assert "warning" in result.get("notes", "").lower()
```

def test_priority_inversion_edge_cases():

        "p0_complete": True,

## Test Rationale

These tests address specific edge cases identified through protocol analysis:

1. **Protocol Violation Recovery**: Tests the room's resilience when protocol order is violated (ghost tile scenario).
2. **Empty Input Handling**: Tests input sanitization boundaries (P0 negative space for invalid inputs).
3. **Concurrent State Modification**: Tests thread-safety assumptions in multi-agent environments.
4. **Memory Exhaustion**: Tests resource constraint handling (P0 for system limits).
5. **Priority Inversion**: Tests protocol logic under contradictory constraints.

Each test follows fleet doctrine by first mapping negative space (P0: what could break), then testing safe operation (P1: valid paths), and finally verifying optimization doesn't violate constraints (P2).
