## Reading: DeadbandRoom Preset

I have read the `DeadbandRoom.py` preset file. Here is my factual analysis:

**Purpose:** The DeadbandRoom is a PLATO training room preset that implements the fleet's Deadband Protocol (P0: negative space, P1: safe channels, P2: optimization). It is designed to train agents to avoid catastrophic failure (P0) before seeking efficiency.

**Key Components:**
1. **Room State:** Tracks agent position in a 2D grid representing "action space."
2. **Deadband Map:** A boolean grid where `True` indicates a "rock" (P0 violation zone).
3. **Channel Map:** A derived grid where `False` in deadband indicates a "safe channel" (P1).
4. **Reward Function:** Penalizes P0 violations heavily (-10), gives small reward for P1 movement (+1), and gives larger reward for P2 optimization (+5) when in a safe channel.
5. **Step Logic:** Agents submit actions (up, down, left, right). The room validates against deadband, updates state, and returns reward and done flag.

**Edge-Case Categories Identified:**
- **Boundary Conditions:** Actions at grid edges.
- **Deadband Adjacency:** Moving along the edge of a deadband zone.
- **Channel Isolation:** Safe cells completely surrounded by deadband.
- **Reward Edge Cases:** Consecutive P2 optimizations, mixed P0/P1 steps.
- **State Persistence:** Ensuring state resets correctly between episodes.

## Writing: 5 Additional Edge-Case Tests

Based on the preset code, I will write 5 edge-case tests that extend beyond basic functionality. These tests will be submitted as training tiles.

### Test 1: Boundary Wrap-Around (P0 Violation at Edge)
**Scenario:** Agent at grid edge (e.g., x=0) attempts to move left (out of bounds). The room should treat this as a P0 violation (deadband) because the action is invalid, not wrap around.
**Assertion:** Reward should be -10, episode done flag should be True.

### Test 2: Deadband "Hugging" (P1 Movement Along Barrier)
**Scenario:** Agent is in a safe channel directly adjacent to a deadband cell. Agent moves parallel to the deadband barrier (e.g., right along a row where above is deadband). This is valid P1 movement.
**Assertion:** Reward should be +1, position should update, deadband should not be triggered.

### Test 3: Isolated Safe Cell (No P2 Possible)
**Scenario:** A safe cell exists but all four adjacent cells are deadband. The agent starts in this cell. Any movement action is a P0 violation.
**Assertion:** All four directional actions should result in reward -10 and done=True. The room should not allow "no-op" as an action (not in API).

### Test 4: Consecutive P2 Optimization in Same Channel
**Scenario:** Agent is in a safe channel wide enough for multiple steps. Agent moves twice in the same direction (e.g., right, right). The first move is P1 (+1). The second move, while still in a safe channel, should be P2 optimization (+5) if it advances toward a goal (if goal exists) or if the room defines optimization as repeated safe movement.
**Assertion:** Second step reward should be +5 (if goal-oriented) or +1 (if room defines P2 only for goal proximity). Need to check preset's P2 condition.

### Test 5: State Reset After P0 Violation
**Scenario:** Agent commits a P0 violation (episode done). Room is reset for a new episode. The new episode's starting state should be independent of previous violation (e.g., not retaining penalty or deadband expansion).
**Assertion:** After reset, agent's position should be initial starting point (e.g., (0,0)), reward should be 0, deadband map should be unchanged from original.
