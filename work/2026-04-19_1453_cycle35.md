### Test 1: Deadband Protocol Priority Violation
**Scenario**: Agent attempts to optimize (P2) before establishing safe channel (P1)
**Expected**: Room rejects tile, returns P0 violation error
**Edge Case**: Agent submits P2 optimization tile with no prior P1 establishment

```rust
#[test]
fn test_p2_before_p1_violation() {
    let mut room = DeadbandRoom::new();
    let agent_id = "test_agent_1";
    
    // Agent tries to submit P2 optimization tile without P1 channel
    let p2_tile = Tile {
        agent_id: agent_id.to_string(),
        priority: Priority::P2,
        content: "Optimizing path".to_string(),
        timestamp: Utc::now(),
    };
    
    let result = room.submit_tile(p2_tile);
    assert!(result.is_err());
    assert_eq!(result.unwrap_err(), RoomError::ProtocolViolation);
}
```

### Test 2: Concurrent Deadband Threshold Contention
**Scenario**: Multiple agents simultaneously submit tiles that would push system beyond deadband threshold
**Expected**: Only first tile accepted, subsequent tiles rejected with threshold exceeded error
**Edge Case**: Race condition on threshold checking

```rust
#[test]
fn test_concurrent_threshold_contention() {
    let room = Arc::new(Mutex::new(DeadbandRoom::new()));
    let mut handles = vec![];
    
    for i in 0..5 {
        let room_clone = Arc::clone(&room);
        handles.push(thread::spawn(move || {
            let tile = Tile {
                agent_id: format!("agent_{}", i),
                priority: Priority::P1,
                content: format!("Channel establishment {}", i),
                timestamp: Utc::now(),
            };
            
            let mut room = room_clone.lock().unwrap();
            room.submit_tile(tile)
        }));
    }
    
    let results: Vec<Result<(), RoomError>> = handles
        .into_iter()
        .map(|h| h.join().unwrap())
        .collect();
    
    // Only first should succeed if threshold is 1 concurrent P1
    let success_count = results.iter().filter(|r| r.is_ok()).count();
    assert_eq!(success_count, 1);
}
```

### Test 3: Ghost Tile Priority Inheritance

**Expected**: Room accepts tile with inherited P0 constraints applied
**Edge Case**: Ghost tile has malformed priority field

        agent_id: "live_agent_1".to_string(),
        priority: Priority::P1,
        content: "Finding channel with ghost constraints".to_string(),

}

### Test 4: Priority Rollback Under System Stress
**Scenario**: System memory exceeds 90%, agent attempts P2 optimization
**Expected**: Room forces rollback to P1 or P0, rejects P2 tile
**Edge Case**: System stress detected mid-tile processing

```rust
#[test]
fn test_priority_rollback_under_stress() {
    let mut room = DeadbandRoom::new();
    let agent_id = "stress_test_agent";
    
    // Simulate system stress (high memory usage)
    room.set_system_stress_level(0.95); // 95% memory used
    
    // Agent has established P1 channel previously
    let p1_tile = Tile {
        agent_id: agent_id.to_string(),
        priority: Priority::P1,
        content: "Safe channel established".to_string(),
        timestamp: Utc::now(),
    };
    room.submit_tile(p1_tile).unwrap();
    
    // Now attempt P2 optimization under stress
    let p2_tile = Tile {
        agent_id: agent_id.to_string(),
        priority: Priority::P2,
        content: "Optimizing path under stress".to_string(),
        timestamp: Utc::now(),
    };
    
    let result = room.submit_tile(p2_tile);
    
    // Should either reject or downgrade to P1
    match result {
        Err(RoomError::SystemStressRollback) => {
            // Expected rejection with rollback instruction
            assert!(room.get_agent_priority(&agent_id) <= Priority::P1);
        }
        Ok(_) => {
            // If accepted, should have been downgraded to P1
            let last_tile = room.get_agent_last_tile(&agent_id);
            assert!(last_tile.priority <= Priority::P1);
        }
        _ => panic!("Unexpected result under system stress"),
    }
}
```

### Test 5: Cross-Room Priority Contamination
**Scenario**: Agent moves from high-priority room to DeadbandRoom with different priority context
**Expected**: DeadbandRoom resets agent to P0, ignoring external room priority
**Edge Case**: Agent brings corrupted priority state from external system

```rust
#[test]
fn test_cross_room_priority_reset() {
    let mut deadband_room = DeadbandRoom::new();
    let agent_id = "mobile_agent_1";
    
    // Simulate agent coming from external room with P2 priority
    let external_state = AgentState {
        id: agent_id.to_string(),
        current_priority: Priority::P2, // External room allowed P2
        external_context: Some("optimization_room".to_string()),
    };
    
    // Agent tries to submit P2 tile immediately in DeadbandRoom
    let p2_tile = Tile {
        agent_id: agent_id.to_string(),
        priority: Priority::P2,
        content: "Continuing optimization from other room".to_string(),
        timestamp: Utc::now(),
        external_context: external_state,
    };
    
    let result = deadband_room.submit_tile(p2_tile);
    
    // DeadbandRoom should reset to P0 and reject P2
    assert!(result.is_err());
    assert_eq!(
        deadband_room.get_agent_priority(&agent_id),
        Priority::P0
    );
    
    // Should log cross-room contamination warning
    let logs = deadband_room.get_recent_logs();
    assert!(logs.iter().any(|log| log.contains("cross-room") && log.contains("reset")));
}
```
