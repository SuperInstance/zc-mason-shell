## 5 Edge-Case Tests for DeadbandRoom

### Test 1: Empty Tile Submission
**Scenario:** Agent submits a tile with empty content (zero-length string or whitespace only).
**Edge Case:** The protocol should handle this as P0 negative space - reject the tile rather than crash.
**Assertion:** Room should return appropriate error code without processing.

### Test 2: Maximum Protocol Violation Chain
**Scenario:** Agent submits 10 consecutive tiles that violate P0 constraints.
**Edge Case:** Testing the room's resilience to repeated protocol violations.
**Assertion:** Room should maintain state and properly reject all violations, not just the first.

### Test 3: Concurrent Tile Processing
**Scenario:** Simulate 5 agents submitting tiles simultaneously.
**Edge Case:** DeadbandRoom's thread safety and state isolation between agents.
**Assertion:** Each agent's protocol state should remain isolated; no cross-contamination.

### Test 4: P2 Optimization with Insufficient P1 History
**Scenario:** Agent attempts P2 optimization after only 1 successful P1 tile.
**Edge Case:** Premature optimization without sufficient safe channel data.
**Assertion:** Room should reject P2 tile and require minimum P1 history (configurable threshold).

### Test 5: Protocol Reset After Extended Inactivity
**Scenario:** Agent submits valid P0 tile after 24 hours of inactivity.
**Edge Case:** Protocol state persistence and timeout handling.
**Assertion:** Room should either maintain previous state or implement timeout reset logic.
